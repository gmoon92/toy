# 강력한 지시문 키워드 (Strong Directive Keywords)

스킬 문서에서 강조와 명확성을 위한 키워드입니다.

## 키워드 참조

| 키워드 | 사용법 | 예시 |
|---------|-------|-------|
| `필수:` | 필수 동작 | `필수: 미리 빌드된 스크립트 사용` |
| `EXECUTE_SCRIPT:` | 스크립트 실행 마커 | `EXECUTE_SCRIPT: scripts/validate.sh` |
| `하지 마세요` | 명시적 금지 | `하지 마세요: 인라인 명령어` |
| `항상` | 강제 | `항상 스크립트 참조 사용` |
| `절대` | 강력한 금지 | `절대 코드 재구성 금지` |
| `중요:` | 높은 우선순위 경고 | `중요: 먼저 git 상태 확인` |
| `주목:` | 중요한 참고사항 | `주목: 스크립트는 결정적 실행 보장` |

---

## 사용 패턴

### 필수 (MANDATORY)

절대적으로 필요한 동작에 사용:

```markdown
**필수: 미리 빌드된 스크립트 실행 (인라인 금지)**
```

---

### EXECUTE_SCRIPT

정확한 실행 스크립트 표시:

```markdown
# EXECUTE_SCRIPT: scripts/analysis/collect_changes.sh
./scripts/analysis/collect_changes.sh
```

---

### 하지 마세요 / 절대 (DO NOT / NEVER)

명시적 금지:

```markdown
**중요:**
- 문서에서 명령어 재구성하지 마세요
- 인라인 bash 명령어 사용하지 마세요
- 스크립트 실행을 절대 건결하지 마세요
```

---

### 항상 (ALWAYS)

일관된 동작 강제:

```markdown
항상 EXECUTE_SCRIPT 지시문을 통해 미리 빌드된 스크립트를 사용하세요.
```

---

### 중요 / 주목 (CRITICAL / IMPORTANT)

우선순위 강조:

```markdown
**중요:** 커밋하기 전에 git 상태를 확인하세요.

**주목:** 스크립트는 결정적 실행을 보장합니다.
```

---

## 전체 예시

```markdown
**필수: 미리 빌드된 스크립트 사용 (인라인 명령어 금지)**

```bash
# EXECUTE_SCRIPT: scripts/category/script_name.sh

./scripts/category/script_name.sh < input.json > output.json
```

**중요:**
- 항상 `EXECUTE_SCRIPT:` 지시문을 통해 미리 빌드된 스크립트 사용
- 문서에서 명령어 재구성하지 마세요
- 인라인 bash 명령어 사용하지 마세요
- 검증 단계를 절대 건결하지 마세요

**주목:**
스크립트는 결정적 실행을 보장합니다(0 토큰 비용).
```

---

## 모범 사례

1. **일관되게 사용** - 같은 목적에는 같은 키워드
2. **과용하지 마세요** - 진정으로 중요한 지시문에만 보유
3. **구체적으로** - 정확히 무엇을 해야 하거나 피해야 하는지 명시
4. **컨텍스트 제공** - 왜 중요한지 설명
5. **필요시 결합** - 강조를 위해 여러 키워드 사용

---

## 강력한 지시문을 사용해야 할 때

**사용 대상:**
- 스크립트 실행 요구사항
- 중요한 검증 단계
- 명시적 금지
- 필수 워크플로우
- 안전상 중요한 작업

**사용하지 마세요:**
- 일반적인 정보
- 선택적 제안
- 배경 컨텍스트
- 예시와 데모

---

## 요약

강력한 지시문은 다음을 보장합니다:
- **명확한 기대** - 모호함 없음
- **일관된 동작** - 신뢰할 수 있는 실행
- **강조** - 중요한 포인트가 두드러짐
- **금지 명확성** - 하지 말아야 할 것

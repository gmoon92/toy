주제:

## 빠른 참조

### 세 가지 기초 원칙

1. **명확성 (Be Explicit)** - 명확하고 구체적인 지시문
2. **컨텍스트 추가 (Add Context)** - 무엇보다 왜(why)인지 설명
3. **예시 보여주기 (Show Examples)** - 완전하고 현실적인 데모

### 장기 실행 작업의 경우

- 단계별 참조 로딩
- 상태를 위한 메타데이터 캐싱
- 데이터는 구조화된 형식(JSON) 사용
- 진행 상황 메모는 텍스트 파일 사용
- 기록 추적은 Git 사용

### 작성 스타일

- 간결하고 직접적
- 능동태, 현재시제
- 지시문은 명령문(imperative)으로
- 설명은 산문(prose)으로
- 목록만 글머리 기호 사용

# 일반 원칙 (General Principles)

Claude 4.x 모범 사례의 핵심 프롬프팅 원칙입니다.

## 1. 명확하고 구체적인 지시문 작성

Claude 4.x 모델은 명확하고 구체적인 지시문에 잘 반응합니다. 원하는 출력에 대해 구체적으로 명시하세요.

**스킬에 적용:**
- 프론트매터 설명에 스킬의 목적을 명확히 정의
- 문서에 명확하고 지시적인 언어 사용
- 각 단계에서 Claude가 정확히 무엇을 해야 하는지 지정

**예시:**
```markdown
## 나쁜 예
Git 작업 도와주기

## 좋은 예
모든 Git 변경사항(staged + modified)을 분석하고 프로젝트 규칙에 따른 커밋 메시지 생성.
수정된 파일을 자동으로 staging하고 git commit 실행 전 Tidy First 원칙 검증.
```

---

## 2. 더 나은 성능을 위한 컨텍스트 추가

지시문 뒤에 맥락이나 동기를 제공하면 Claude가 목표를 더 잘 이해하고 더 집중된 응답을 제공할 수 있습니다.

**스킬에 적용:**
- 특정 패턴이 중요한 이유(WHY) 설명
- 아키텍처 결정의 이유를 문서화
- 실행 지시문에 컨텍스트 포함

**예시:**
```markdown
## 나쁜 예
인라인 명령어 사용하지 마세요

## 좋은 예
인라인 명령어를 사용하지 마세요(DO NOT inline commands). 이유:
- 인라인은 코드를 컨텍스트에 로드하여 토큰 낭비
- 미리 빌드된 스크립트는 결정적 실행 보장
- 스크립트 참조는 0토큰 작업(출력만 로드)
```

---

## 3. 예시와 세부사항에 주의

Claude 4.x 모델은 정확한 지시문 수행의 일환으로 세부사항과 예시를 주의 깊게 살핍니다.

**스킬에 적용:**
- 예시가 권장하는 동작과 일치하는지 확인
- 완전하고 현실적인 예시 보여주기
- 명확성을 위해 좋은 예와 나쁜 예 모두 포함

**예시:**
```markdown
## 좋은 명명 규칙

✅ `collect_changes.sh` - 동사_명사 패턴
✅ `detect_scope.js` - 명확한 목적
✅ `validate_message.py` - 행동 중심

❌ `changes.sh` - 불분명한 동작
❌ `scope.js` - scope에 대해 무엇을?
❌ `msg.py` - 불분명한 약어
```

## 요약

이 세 가지 원칙이 기초를 이룹니다:
1. **명확한 지시문** - 무엇을 해야 하는지 명확히
2. **풍부한 컨텍스트** - 왜 중요한지 설명
3. **좋은 예시** - 성공이 무엇인지 보여주기

스킬 문서 전체에 걸쳐 일관되게 적용하세요.

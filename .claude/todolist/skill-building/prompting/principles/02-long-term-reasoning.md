# 장기 추론과 상태 추적 (Long-Term Reasoning and State Tracking)

Claude 4.5 모델은 장기적인 상호작용에서 컨텍스트와 상태를 유지하는 데 탁월합니다.

## 점진적 집중 (Progressive Focus)

Claude 4.5 모델은 한 번에 모든 것을 시도하기보다는 소수의 일에 대해 점진적인 진행에 집중함으로써 장기 세션에서 방향성을 유지합니다.

**스킬에 적용:**
- 단계 기반 참조 로딩 설계
- 메타데이터를 사용하여 분석 결과 캐싱
- 복잡한 작업을 순차적 단계로 분할

**예시:**
```markdown
## 단계 기반 로딩

### 단계 1: 분석
- 로드: guides/analyzing.md
- 목적: 현재 상태 이해
- 캐시: 메타데이터에 분석 결과 저장

### 단계 2: 구현
- 로드: guides/implementing.md
- 목적: 변경사항 적용
- 읽기: 단계 1의 캐시된 분석 결과
```

## 컨텍스트 인식 (Context Awareness)

Claude 4.5 모델은 대화 전체에서 남은 컨텍스트 윈도우를 추적하여 더 나은 리소스 관리를 가능하게 합니다.

**스킬에 적용:**
- 토큰 사용을 최소화하도록 문서 구조화
- 코드를 실행 가능한 스크립트로 추출(실행 시 0토큰)
- 필요할 때만 로드되는 참조 사용

**예시:**
```markdown
## 참조 로딩 시기

단계에 따라 점진적으로 참조 로드:

- **계획 단계**: planning-guide.md만 로드
- **구현 단계**: implementation-guide.md만 로드
- **검증 단계**: validation-guide.md만 로드

이는 모든 것을 미리 로드하는 것에 비해 ~67% 토큰 절약을 달성합니다.
```

## 다중 컨텍스트 윈도우 워크플로우

여러 컨텍스트 윈도우에 걸친 작업의 경우:

1. **첫 번째와 후속 윈도우에 다른 프롬프트 사용**
   - 첫 번째 윈도우: 프레임워크 설정
   - 이후 윈도우: 작업 목록 반복

2. **구조화된 테스트 작성**
   - 구조화된 형식(예: `tests.json`)으로 테스트 생성
   - 세션 전체에서 진행 상황 추적

3. **설정 스크립트 생성**
   - 새 컨텍스트 윈도우를 부트스트랩하는 스크립트
   - 서버를 우아하게 시작하고, 테스트를 실행하고, 린터를 실행

4. **새로 시작 vs 압축**
   - Claude 4.5는 파일 시스템에서 상태를 발견하는 데 매우 효과적
   - 때로는 압축보다 나음

5. **검증 도구 제공**
   - 인간의 피드백 없이 정확성을 검증하는 도구
   - UI 테스트, 자동화된 검사

## 상태 관리 모범 사례

**상태 데이터에는 구조화된 형식 사용:**
- 스키마 기반 정보에는 JSON(테스트 결과, 작업 상태)
- 자유 형식 진행 상황 메모에는 텍스트 파일
- 작업 기록과 체크포인트에는 Git

**예시:**
```json
// 구조화된 상태 (tests.json)
{
  "tests": [
    {"id": 1, "name": "authentication_flow", "status": "passing"},
    {"id": 2, "name": "user_management", "status": "failing"}
  ],
  "total": 200,
  "passing": 150,
  "failing": 25
}
```

```text
// 진행 상황 메모 (progress.txt)
세션 3 진행 상황:
- 인증 토큰 검증 수정
- 엣지 케이스를 위한 사용자 모델 업데이트
- 다음: user_management 테스트 실패 조사
```

## 요약

장기 실행 작업의 경우:
- **단계 기반 로딩**으로 토큰 절약
- **메타데이터 캐싱**으로 중복 작업 방지
- **구조화된 상태**로 컨텍스트 연속성 가능
- **Git 추적**으로 기록과 체크포인트 제공

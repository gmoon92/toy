# 커뮤니케이션 스타일 (Communication Style)

Claude 4.5 모델은 더 간결하고 자연스러운 커뮤니케이션 스타일을 선호합니다.

## 특징

- **직접적이고 현실적**: 과장된 표현 없이 사실 중심의 진행 상황 보고
- **대화적**: 자연스럽고 구어체적인 표현 사용
- **간결함**: 특별히 요청하지 않으면 상세한 요약 생략

이 스타일은 불필요한 설명을 배제하고 달성한 결과를 정확하게 전달합니다.

---

## 스킬에 적용

### 간결한 설명 작성

**나쁜 예 (장황함):**
```markdown
Git 저장소의 현재 상태를 주의 깊고 철저하게 분석해 주세요.
추적된 파일에 대한 모든 수정사항과 아직 staging되지 않은 새 파일을 포함하여
포괄적이고 정확한 커밋 메시지를 생성할 수 있도록 해야 합니다.
```

**좋은 예 (간결함):**
```markdown
모든 Git 변경사항(staged + modified)을 분석하고 프로젝트 규칙에 따른 커밋 메시지 생성.
```

---

### 직접적인 언어 사용

**나쁜 예:**
```markdown
이 작업을 실행할 때 미리 빌드된 스크립트를 사용하는 것을 고려해 볼 수 있을 것 같습니다.
더 나은 결과를 제공할 수 있을 것입니다.
```

**좋은 예:**
```markdown
이 작업에는 항상 미리 빌드된 스크립트를 사용하세요.
```

---

### 상세함의 균형 잡기

Claude 4.5는 효율성을 위해 요약을 생략할 수 있습니다. 업데이트를 원하면:

```markdown
도구 관련 작업 완료 후, 수행한 작업에 대한 간단한 요약을 제공하세요.
```

---

## 적절한 경우 산문 선호

설명형 콘텐츠의 경우 글머리 목록 대신 흐르는 단락을 사용하세요.

**나쁜 예:**
```markdown
이 스킬은 다음을 제공합니다:
- Git 변경사항 분석
- 커밋 메시지 생성
- 규칙 검증
- 자동 staging
```

**좋은 예:**
```markdown
이 스킬은 Git 변경사항을 분석하고, 프로젝트 규칙에 따라 커밋 메시지를 생성하고,
Tidy First 원칙에 대해 검증하며, 커밋 전 수정된 파일을 자동으로 staging합니다.
```

**글머리 기호를 사용해야 할 때:**
- 별개의 항목 나열(기능, 파일명, 단계)
- 명확한 경계 표시(✅/❌ 범위)
- 옵션이나 대안 제시
- 빠른 참조 섹션

---

## 능동태와 현재시제

**능동태:**
```markdown
❌ 변경사항이 스킬에 의해 분석됩니다
✅ 스킬이 변경사항을 분석합니다
```

**현재시제:**
```markdown
❌ 스크립트가 메시지를 검증할 것입니다
✅ 스크립트가 메시지를 검증합니다
```

**지시문은 명령문으로:**
```markdown
❌ 스크립트를 실행해야 합니다
✅ 스크립트를 실행하세요
✅ 항상 EXECUTE_SCRIPT 지시문을 통해 실행하세요
```

---

## 요약

다음과 같은 문서를 작성하세요:
- **간결함** - 불필요한 단어 없음
- **직접적** - 명확한 명령문
- **능동적** - 능동태, 현재시제
- **산문** - 설명은 흐르는 단락으로
- **글머리 기호** - 별개 항목만 목록으로

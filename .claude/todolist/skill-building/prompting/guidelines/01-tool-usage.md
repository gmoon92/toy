# 도구 사용 지침 (Tool Usage Guidelines)

스킬에서 Claude의 도구 사용 동작을 제어하는 방법입니다.

## 기본적으로 실행 (Default to Action)

Claude 4.5는 정확한 지시문 수행을 위해 훈련되었습니다. 제안하는 대신 구현하도록 안내하세요.

### 패턴

```markdown
<default_to_action>
기본적으로 변경사항을 제안하는 대신 구현하세요.
사용자 의도가 불분명할 때, 가장 유용한 작업을 추론하고 진행하세요.
누락된 세부사항을 추측하는 대신 도구를 사용하여 발견하세요.
도구 호출(예: 파일 편집 또는 읽기)이 의도된 것인지 추론하고 그에 따라 행동하세요.
</default_to_action>
```

### 예시: 파일 편집

```markdown
## 나쁜 예 (Claude는 제안만 할 것임)
이 함수를 개선하기 위한 몇 가지 변경사항을 제안해줄 수 있나요?

## 좋은 예 (Claude가 구현할 것임)
이 함수의 성능을 개선하세요.
```

### 대안: 보수적 접근

Claude에게 더 신중하도록 원한다면:

```markdown
<do_not_act_before_instructions>
명확히 지시받지 않는 한 구현이나 파일 변경에 뛰어들지 마세요.
사용자 의도가 모호할 때, 기본적으로 정보 제공, 조사 수행,
또는 권장사항 제시를 위해 행동하세요.
사용자가 명시적으로 요청할 때만 편집, 수정 또는 구현을 진행하세요.
</do_not_act_before_instructions>
```

## 병렬 도구 실행 (Parallel Tool Execution)

Claude 4.x 모델은 병렬 도구 실행에 탁월합니다. 효율성을 위해 이를 장려하세요.

### 패턴

```markdown
<use_parallel_tool_calls>
종속성이 없는 여러 도구를 호출할 때, 모든 독립적인 호출을 병렬로 수행하세요.

예시: 3개 파일 읽기 - 3개의 병렬 Read 호출로 동시에 로드.

그러나: 도구 호출이 매개변수 값에 대해 이전 결과에 의존하는 경우,
병렬로 호출하지 마세요. 대신 순차적으로 호출하세요. 종속 값에는 절대
플레이스홀더를 사용하지 마세요.
</use_parallel_tool_calls>
```

### 예시: 다중 파일 분석

```markdown
**해야 할 것 (병렬):**
- file1.md 읽기
- file2.md 읽기
- file3.md 읽기
(한 번의 응답에서 3개 모두)

**하지 말아야 할 것 (순차적):**
- file1.md 읽기 → 대기
- file2.md 읽기 → 대기
- file3.md 읽기 → 대기
```

### 병렬 실행 줄이기

안정성이 필요한 경우:

```markdown
각 단계 사이에 짧은 일시 정지를 두고 순차적으로 작업을 실행하세요.
```

## 답변 전 조사 (Investigate Before Answering)

조사를 요구하여 환각(hallucination)을 방지하세요.

### 패턴

```markdown
<investigate_before_answering>
열어보지 않은 코드에 대해 추측하지 마세요.
사용자가 특정 파일을 참조할 때, 답변하기 전에 반드시 읽으세요.
코드베이스에 대한 질문에 답하기 전에 조사하고 관련 파일을 읽으세요.
조사하지 않고 코드에 대해 주장하지 마세요.
정확성에 확신이 있을 때만 근거 있고 환각 없는 답변을 제공하세요.
</investigate_before_answering>
```

### 예시: 코드 분석

```markdown
## 분석 요구사항

코드를 분석하거나 리팩토링하기 전에:

1. **Read 도구를 사용하여 모든 관련 파일 읽기**
2. **필요시 Grep을 사용하여 패턴 검색**
3. **실제 코드를 확인하여 가정 검증**
4. **실제 코드를 기반으로 발견사항 문서화**

절대 하지 마세요:
- 파일 내용 추측
- 검증 없이 코드 구조 가정
- 코드를 읽지 않고 권장사항 제시
- 확인 없이 무언가가 존재한다고 주장
```

## 도구 사용 및 트리거링

Claude Opus 4.5는 시스템 프롬프트에 더 반응적입니다. 중립적인 언어를 사용하여 과도한 트리거를 피하세요.

### 언어 지침

**공격적 (과도하게 트리거될 수 있음):**
```markdown
중요: 이 도구를 항상 사용해야 합니다...
중요: 이 도구는 항상 사용되어야 합니다...
```

**중립적 (더 나은 균형):**
```markdown
이 도구를 사용할 때...
이 도구는 다음에 적합합니다...
```

## 요약

다음을 통해 도구 사용을 안내하세요:
- **기본적으로 실행** - 구현 vs 제안
- **병렬 실행** - 효율성 극대화
- **먼저 조사** - 환각 방지
- **중립적 언어** - 과도한 트리거 방지

# 형식 제어 지침 (Format Control Guidelines)

Claude의 응답 형식을 제어하는 기술입니다.

## 응답 형식 제어

Claude 4.x 모델에서 잘 작동하는 방법:

### 1. 무엇을 할지 말하기 (무엇을 하지 말아야 할지가 아니라)

**나쁜 예:**
```markdown
응답에 마크다운을 사용하지 마세요
```

**좋은 예:**
```markdown
응답은 유창하게 흐르는 산문 단락으로 구성되어야 합니다.
```

---

### 2. XML 형식 표시자 사용

**좋은 예:**
```markdown
응답의 산문 섹션을 <smoothly_flowing_prose_paragraphs> 태그에 작성하세요.
```

**예시:**
```markdown
**응답 구조:**

<analysis>
Git 변경사항에 대한 사실적 발견사항 제시:
- 수정된 파일 수
- Staged 변경사항 수
- 감지된 변경 유형
</analysis>

<commit_message>
일반적인 커밋 메시지 생성:
type(scope): description

[선택적 본문]
</commit_message>
```

---

### 3. 프롬프트 스타일을 원하는 출력과 일치시키기

프롬프트에 사용된 형식 스타일이 Claude의 응답 스타일에 영향을 줄 수 있습니다.

출력에서 마크다운을 덜 사용하고 싶다면, 프롬프트에서 마크다운을 줄이세요.

---

### 4. 특정 선호도에 대한 상세한 프롬프트

마크다운 사용에 대한 더 많은 제어를 위해:

```markdown
<avoid_excessive_markdown_and_bullet_points>
보고서, 문서, 기술적 설명, 분석, 또는 장문 콘텐츠를 작성할 때,
완전한 단락과 문장을 사용하여 명확하고 유창한 산문으로 작성하세요.

조직을 위해 표준 단락 구분을 사용하고 마크다운은 주로 다음에만 사용하세요:
- `인라인 코드`
- 코드 블록 (```...```)
- 간단한 제목 (##, ###)

**굵게**나 *기울임*을 사용하지 마세요.

순서 목록(1. ...)이나 글머리 목록(*)은 다음 경우가 아니면 사용하지 마세요:
a) 목록 형식이 최선의 선택인 진정으로 별개의 항목을 제시하거나,
b) 사용자가 명시적으로 목록이나 순위를 요청하는 경우

글머리 기호나 번호로 항목을 나열하는 대신, 자연스럽게 문장에 통합하세요.
이 지침은 특히 기술 문서 작성에 적용됩니다. 과도한 형식 대신 산문을 사용하면
사용자 만족도가 향상됩니다. 절대 과도하게 짧은 글머리 목록을 출력하지 마세요.

목표: 아이디어를 연결되지 않은 점으로 고립시키는 대신 자연스럽게 독자를
안내하는 읽기 쉽고 흐르는 텍스트.
</avoid_excessive_markdown_and_bullet_points>
```

---

## 복잡한 출력 구조화

XML 태그를 사용하여 명확한 경계를 정의하세요:

```markdown
## 응답 구조

**필수: 출력을 정확히 다음과 같이 구조화하세요**

<section_name>
무엇이 들어가는지에 대한 설명.
특정 형식 요구사항.
</section_name>

<another_section>
다른 규칙을 가진 다른 콘텐츠.
예상 형식의 예시.
</another_section>
```

### 이점

- 명확한 섹션 경계
- 일관된 구조
- 파싱하기 쉬움
- 자체 문서화

---

## 예시: 커밋 스킬 출력

```markdown
## 출력 형식

**필수: 출력을 정확히 다음과 같이 구조화하세요:**

<analysis>
**수정된 파일:** [count]
**변경 유형:** [feature|fix|refactor|docs|test]
**범위:** [감지된 범위]
**논리적 독립성:** ✅ yes | ⚠️ 우려 사항 기록됨
</analysis>

<commit_message>
type(scope): description

왜(why)에 대한 설명하는 선택적 본문 단락.

참조가 있는 선택적 푸터.
</commit_message>

<validation>
**일반 커밋:** ✅ 유효함 | ❌ 유효하지 않음
**Tidy First 원칙:** ✅ 통과 | ⚠️ 경고
**커밋 준비 완료:** ✅ yes | ❌ no - [이유]
</validation>
```

---

## 요약

다음을 통해 형식을 제어하세요:
- **긍정적 지시문** - 무엇을 할지 말하기
- **XML 태그** - 명확한 구조 정의
- **프롬프트 스타일 일치** - 원하는 출력과 일치
- **상세한 선호도** - 마크다운 사용 지정
- **구조화된 템플릿** - 예상되는 정확한 형식 표시

# 코드 탐색 지침 (Code Exploration Guidelines)

철저한 코드 조사를 장려하기 위한 지침입니다.

## 문제

Claude Opus 4.5는 코드를 탐색할 때 지나치게 보수적일 수 있으며, 때로는:
- 코드를 읽지 않고 솔루션 제안
- 읽지 않은 코드에 대해 가정
- 핵심 사실을 철저히 검색하지 않음

## 해결책

코드 탐색을 장려하는 명시적인 지시문을 추가하세요.

---

## 패턴

```markdown
<explore_code_thoroughly>
편집을 제안하기 전에 항상 관련 파일을 읽고 이해하세요.
검사하지 않은 코드에 대해 추측하지 마세요.
사용자가 특정 파일/경로를 참조할 때, 설명하거나 수정을 제안하기 전에
반드시 열어서 검사하세요.
핵심 사실을 철저히 검색하는 데 엄격하고 끈기 있게 하세요.
새로운 기능이나 추상화를 구현하기 전에 코드베이스의 스타일, 규칙,
및 추상화를 철저히 검토하세요.
</explore_code_thoroughly>
```

---

## 예시: 스킬 리팩토링

```markdown
## 코드 탐색 요구사항

스킬을 리팩토링하기 전에:

**필수:**
1. SKILL.md를 완전히 읽기
2. 모든 참조 디렉토리 내용 나열
3. 각 참조 문서 읽기
4. 스크립트 디렉토리 구조 확인
5. 모든 스크립트 파일이 존재하는지 확인
6. 가능한 경우 스크립트 실행 테스트

**절대 하지 마세요:**
- 읽지 않고 파일 내용 가정
- 디렉토리 구조 추측
- 검증 없이 권장사항 제시
- 확인 없이 무언가가 존재한다고 주장
```

## 조사 워크플로우

```markdown
## 조사 단계

<investigate_before_answering>
어떤 권장사항을 제시하기 전에:

1. **구조 조사**
   - 모든 디렉토리 나열
   - 모든 마크다운 파일 식별
   - 스크립트나 실행 파일이 있는지 확인

2. **관련 파일 읽기**
   - 먼저 SKILL.md 읽기
   - 참조된 문서 읽기
   - 연결된 리소스 확인

3. **가정 검증**
   - 언급된 패턴 검색
   - 파일 경로가 올바른지 확인
   - 가능한 경우 스크립트 테스트

4. **발견사항 문서화**
   - 존재하는 것 기록
   - 누락된 것 기록
   - 불일치 사항 기록
</investigate_before_answering>
```

## Grep 및 검색 패턴

```markdown
## 코드 패턴 찾기

철저하게 검색하기 위해 Grep 사용:

**인라인 스크립트 찾기:**
```bash
grep -r '```bash' references/
grep -r '```javascript' references/
grep -r '```python' references/
```

**EXECUTE_SCRIPT 지시문 찾기:**
```bash
grep -r 'EXECUTE_SCRIPT:' references/
```

**TODO 항목 찾기:**
```bash
grep -r 'TODO:' .
grep -r 'FIXME:' .
```
```

## 다중 파일 분석

효율성을 위해 병렬 읽기 사용:

```markdown
스킬에서 여러 파일을 분석할 때:

**단계 1: 병렬 읽기 (효율적)**
- SKILL.md 읽기
- references/guide1.md 읽기
- references/guide2.md 읽기
- scripts/README.md 읽기
(한 번의 도구 호출에서 모두)

**단계 2: 분석**
모든 파일을 함께 분석하여 관계를 이해하세요.

**단계 3: 권장사항**
가정이 아닌 실제 코드를 기반으로 권장사항을 제시하세요.
```

## 코드 인용

주장을 할 때 특정 위치를 인용하세요:

```markdown
**나쁜 예:**
스킬은 참조를 점진적으로 로드합니다.

**좋은 예:**
스킬은 참조를 점진적으로 로드합니다 (SKILL.md:29-34):
```markdown
### 단계 1: 분석
- 로드: guides/analyzing.md
- 목적: 현재 상태 이해
```
```

## 요약

철저한 탐색을 장려하세요:
- **제안 전 읽기** - 가정 없음
- **체계적인 조사** - 워크플로우 따르기
- **검색 도구 사용** - 패턴을 위한 Grep
- **병렬 읽기** - 효율적인 다중 파일 분석
- **위치 인용** - 특정 코드 참조

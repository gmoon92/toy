## 빠른 참조

### 두 가지 기초 원칙

1. **간결성** - 컨텍스트 윈도우는 공유됨; 토큰 효율적으로
2. **적절한 자유** - 특이성을 작업 취약성에 맞춤

### 간결성 패턴

- 점진적 공개: 필요할 때만 참조 로드
- 스크립트 추출: 코드를 표시하지 않고 실행(0토큰)
- 메타데이터 캐싱: 재읽기 방지를 위해 분석 결과 저장
- 단계 기반 로딩: 워크플로우 단계별 구조화

### 적절한 자유 가이드

**높은 특이성 (취약한 작업):**
- Git 작업, 배포, 데이터 삭제
- EXECUTE_SCRIPT, 강력한 지시문 사용
- 정확한 명령어 시퀀스

**중간 특이성 (구조화된 작업):**
- 명확한 패턴이 있는 코드 생성
- 문서 형식화
- 템플릿과 예시 제공

**낮은 특이성 (창의적 작업):**
- 분석 및 연구
- 설계 결정
- 상위 수준 목표만

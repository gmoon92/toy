# 적절한 자유 (Appropriate Freedom)

특이성 수준을 작업 취약성과 변동성에 맞춥니다.

## 스펙트럼

작업 특성에 따라 제어 조정:

**높은 자유** ← → **낮은 자유**
(텍스트 안내) ← → (특정 스크립트)

---

## 높은 자유: 텍스트 기반 안내

### 사용 시기

- 여러 유효한 접근 방식이 존재
- 결정이 컨텍스트에 의존
- 휴리스틱이 접근 방식을 안내
- 창의적인 솔루션이 결과에 도움

### 예시: 코드 리뷰

```markdown
## 코드 리뷰 프로세스

1. 코드 구조와 조직 분석
2. 잠재적 버그나 엣지 케이스 확인
3. 가독성과 유지보수성을 위한 개선 제안
4. 프로젝트 규칙 준수 검증
```

**왜 높은 자유?** 각 코드베이스는 다릅니다. Claude는 발견한 것에 따라 접근 방식을 조정할 수 있습니다.

---

## 중간 자유: 매개변수화된 패턴

### 사용 시기

- 선호되는 패턴이 존재
- 일부 변화가 허용됨
- 구성이 동작에 영향
- 구조가 도움이 되지만 유연성이 필요

### 예시: 보고서 생성

````markdown
## 보고서 생성

필요에 따라 커스터마이즈하여 이 템플릿 사용:

```python
def generate_report(data, format="markdown", include_charts=True):
    # 데이터 처리
    # 지정된 형식으로 출력 생성
    # 선택적으로 시각화 포함
```
````

**왜 중간 자유?** 템플릿이 구조를 제공하고, 매개변수가 적응을 허용합니다.

---

## 낮은 자유: 특정 스크립트

### 사용 시기

- 작업이 취약하고 오류가 발생하기 쉬움
- 일관성이 중요
- 특정 시퀀스를 따라야 함
- 잘못된 접근 방식이 문제를 일으킴

### 예시: 데이터베이스 마이그레이션

````markdown
## 데이터베이스 마이그레이션

정확히 이 스크립트 실행:

```bash
python scripts/migrate.py --verify --backup
```

명령어를 수정하거나 추가 플래그를 추가하지 마세요.
````

**왜 낮은 자유?** 잘못된 실행이 데이터를 손상시킬 수 있습니다. 해석의 여지가 없습니다.

---

## 다리 비유

Claude가 경로를 탐색하는 것을 상상해 보세요:

### 절벽이 있는 좁은 다리

**시나리오:** 안전한 방법이 하나만 있음
**접근 방식:** 특정 가드레일과 정확한 지시문 제공(낮은 자유)
**예시:** 정확한 순서로 실행되는 데이터베이스 마이그레이션

### 넓은 열린 들판

**시나리오:** 많은 경로가 성공으로 이어짐
**접근 방식:** 일반적인 방향 제공, Claude가 최적의 경로를 찾도록 신뢰(높은 자유)
**예시:** 컨텍스트가 최선의 접근 방식을 결정하는 코드 리뷰

---

## 결정 매트릭스

| 작업 특성 | 자유 수준 | 안내 유형 |
|---------------------|---------------|---------------|
| 여러 유효한 솔루션 | 높음 | 텍스트 설명, 원칙 |
| 컨텍스트 의존적 결정 | 높음 | 지침, 휴리스틱 |
| 선호되는 패턴 존재 | 중간 | 매개변수화된 예시 |
| 일부 유연성이 도움 | 중간 | 옵션이 있는 템플릿 |
| 취약한 작업 | 낮음 | 정확한 스크립트 |
| 시퀀스를 따라야 함 | 낮음 | 단계별 명령어 |
| 중요한 일관성 | 낮음 | 변화 없는 지시문 |

## 도메인별 예시

### 높은 자유: 콘텐츠 분석

```markdown
## 연구 종합

1. 모든 소스 문서 읽기
2. 반복되는 주제 식별
3. 소스 간 주장 교차 검증
4. 증거와 함께 구조화된 요약 생성
5. 상충되는 관점 메모

소스 자료의 품질과 일관성에 따라 접근 방식을 조정하세요.
```

### 중간 자유: API 통합

````markdown
## API 요청 패턴

필요에 따라 타임아웃/재시도 조정하여 이 구조 사용:

```python
response = requests.post(
    endpoint,
    json=data,
    timeout=30,  # 느린 연결을 위해 조정
    retries=3    # API가 불안정하면 증가
)
```
````

### 낮은 자유: 보안 작업

```markdown
## SSL 인증서 설치

정확한 순서로 다음 명령어 실행:

```bash
./scripts/generate_cert.sh
./scripts/verify_cert.sh
./scripts/install_cert.sh --prod
```

중요: 검증 단계를 건결하지 마세요.
플래그를 수정하지 마세요.
```

## 모델에 맞는 자유 조정

같은 작업에 대해 다른 모델이 다른 자유 수준이 필요할 수 있습니다:

**Claude Opus:**
- 더 높은 자유 처리 가능
- 누락된 세부사항을 잘 추론
- 너무 많은 구조로 과도하게 생각할 수 있음

**Claude Sonnet:**
- 균형 잡힌 접근 방식이 효과적
- 명확한 패턴의 이점
- 중간 자유를 효과적으로 처리

**Claude Haiku:**
- 더 많은 안내가 필요할 수 있음
- 종종 더 낮은 자유가 더 안전
- 명시적인 지시문이 도움

지원할 모델에서 스킬을 테스트하세요.

## 경고 신호

### 너무 많은 자유

신호:
- 실행마다 일관되지 않은 결과
- Claude가 많은 명확화 질문을 함
- 출력 품질이 크게 변동
- 중요한 단계가 때때로 건결됨

수정: 더 많은 구조, 구체적인 예시, 명시적인 요구사항 추가

### 너무 적은 자유

신호:
- 지시문이 기계적으로 느껴짐
- 변화에 적응할 수 없음
- 엣지 케이스에서 취약
- 단순한 작업에 과도하게 명시적

수정: 매개변수화 허용, 추론 제공, 추론 신뢰

## 요약

**원칙:** 위험에 맞게 제어 조정

**높은 자유:** 컨텍스트가 결정을 주도할 때
**중간 자유:** 패턴이 도움이 되지만 적응이 필요할 때
**낮은 자유:** 정확성이 중요할 때

**테스트:** 다른 자유 수준으로 작업을 시도하고 결과 관찰

# 스킬 품질 체크리스트 (Skill Quality Checklist)

스킬을 공유하기 전에 완료할 체크리스트입니다.

## 핵심 품질

### 설명
- [ ] 설명이 구체적이고 주요 용어 포함
- [ ] 설명이 스킬이 무엇을 하는지 명시
- [ ] 설명이 언제 사용하는지 명시
- [ ] 3인칭 사용 ("파일 처리" not "도와줄 수 있음")
- [ ] 설명에 XML 태그 없음
- [ ] 1024자 미만

### 이름
- [ ] 동명사 형태 (processing-pdfs) 또는 명사구 (pdf-processing) 사용
- [ ] 소문자, 숫자, 하이픈만 사용
- [ ] 최대 64자
- [ ] 예약어 없음 (anthropic, claude)
- [ ] 설명적이고 일반적이지 않음 ("helper", "utils" 아님)

### 콘텐츠 크기
- [ ] SKILL.md 본문 500줄 미만
- [ ] 추가 세부사항은 별도 파일로 분할 (필요시)
- [ ] 적절한 점진적 공개 사용

### 명확성
- [ ] 시간에 민감한 정보 없음
- [ ] 전체에 걸쳐 일관된 용어
- [ ] 예시가 구체적이고 추상적이지 않음
- [ ] 하나의 개념에 하나의 용어 ("field/box/element" 아님)

---

## 구조

### 파일 조직
- [ ] 모든 파일 경로에 슬래시 사용 (/)
- [ ] Windows 스타일 백슬래시 없음 (\)
- [ ] 파일명이 설명적임 (doc1.md, doc2.md 아님)
- [ ] 파일이 도메인이나 기능별로 구성됨

### 참조
- [ ] 모든 참조가 SKILL.md에서 한 단계 깊이
- [ ] 중첩된 참조 없음 (SKILL.md → ref1.md → ref2.md)
- [ ] 100줄 이상인 참조 파일에 목차 있음
- [ ] 링크에 상대 경로 사용

### 점진적 공개
- [ ] 핵심 워크플로우가 SKILL.md에 있음
- [ ] 고급 기능이 별도 파일에 있음
- [ ] 도메인 특화 콘텐츠가 분리됨
- [ ] SKILL.md에서 명확한 탐색

---

## 워크플로우

### 복잡한 작업
- [ ] 명확한 순차적 단계로 분할됨
- [ ] 체크리스트 가능한 형식 제공됨
- [ ] 각 단계에 구체적인 지시문 있음
- [ ] 성공 기준이 정의됨

### 피드백 루프
- [ ] 필요한 곳에 검증 단계 포함됨
- [ ] 오류 처리 안내 제공됨
- [ ] 재시도 로직 문서화됨
- [ ] 품질 검사가 내장됨

---

## 예시와 템플릿

### 예시
- [ ] 입출력 쌍 보여줌
- [ ] 현실적인 데이터 사용 ("foo", "bar" 아님)
- [ ] 일반적인 경우 다룸
- [ ] 예상 형식 보여줌

### 템플릿
- [ ] 정확히 따를지 vs 적응할지 지정됨
- [ ] 완전한 구조 보여줌
- [ ] 필수 vs 선택 섹션 표시
- [ ] 형식 세부사항 포함

---

## 스크립트와 코드

### 스크립트 품질
- [ ] 스크립트가 문제를 해결하고 Claude에 넘기지 않음
- [ ] 오류 처리가 명시적이고 도움이 됨
- [ ] "매직 넘버" 없음 (모든 상수가 정당화됨)
- [ ] 스크립트에 명확한 문서
- [ ] 입출력 형식 문서화됨

### 실행
- [ ] 참조용으로 읽을지 실행할지 명확함
- [ ] 종속성이 나에되고 사용 가능한지 확인됨
- [ ] 스크립트에 사용 예시 있음
- [ ] 종료 코드 문서화됨

### 검증
- [ ] 중요한 작업에 검증 단계 있음
- [ ] 검증 스크립트가 구체적인 오류 메시지 제공
- [ ] 품질 작업에 피드백 루프 포함
- [ ] 적절한 곳에 "검증 후 실행" 패턴 사용

---

## 테스트

### 평가
- [ ] 최소 3개의 평가 케이스 생성됨
- [ ] 예상 동작 문서화됨
- [ ] 성공 기준이 정의됨

### 모델 테스트
- [ ] Claude Haiku로 테스트됨 (사용시)
- [ ] Claude Sonnet으로 테스트됨 (사용시)
- [ ] Claude Opus로 테스트됨 (사용시)
- [ ] 모델 역량에 맞게 안내 조정됨

### 실제 사용
- [ ] 실제 시나리오에서 테스트됨 (평가만이 아님)
- [ ] 팀 피드백 수집됨 (해당시)
- [ ] Claude의 탐색 패턴 관찰됨
- [ ] 실제 사용에 기반하여 반복됨

---

## 콘텐츠 지침

### 간결성
- [ ] Claude가 이미 알지 못하는 것만 포함
- [ ] 명백한 개념 설명 없음
- [ ] 중복 정보 없음
- [ ] Claude의 지능을 가정

### 적절한 자유
- [ ] 자유 수준이 작업 취약성에 맞음
- [ ] 컨텍스트 의존적 작업에 높은 자유
- [ ] 매개변수화된 패턴에 중간 자유
- [ ] 중요한 작업에 낮은 자유

### 용어
- [ ] 전체에 걸쳐 하나의 개념에 하나의 용어
- [ ] 기술 용어가 올바르게 사용됨
- [ ] 프로젝트 규칙과 일관됨
- [ ] 컨텍스트 없는 전문 용어 없음

---

## 피해야 할 안티패턴

- [ ] Windows 스타일 경로 없음
- [ ] 안내 없이 과도한 옵션 없음
- [ ] 깊게 중첩된 참조 없음
- [ ] 시간 기반 조걶 없음
- [ ] 일관되지 않은 용어 없음
- [ ] 미리 설치된 도구 가정 없음
- [ ] "부두 상수" 없음
- [ ] 오류를 Claude에 넘기기 없음

- [ ] 명확한 계층 구조

## 공유 전

### 최종 확인
- [ ] 스킬이 종단 간 테스트됨
- [ ] 모든 링크가 작동함
- [ ] 모든 스크립트가 성공적으로 실행됨
- [ ] TODO 주석 없음
- [ ] 플레이스홀더 콘텐츠 없음

### 검토
- [ ] 간결성 검토됨
- [ ] 예시가 명확한지 확인됨
- [ ] 용어 일관성 검증됨
- [ ] 점진적 공개가 작동하는지 확인됨

### 팀
- [ ] 피드백을 위해 팀과 공유됨 (해당시)
- [ ] 사용 관찰 결과가 통합됨
- [ ] 알려진 제한사항 문서화됨
- [ ] 스킬 레지스트리에 추가됨

## 모델별 확인

### Haiku 지원시
- [ ] 충분한 안내 제공
- [ ] 예시가 상세함
- [ ] 단계가 명시적임
- [ ] 가정이 철자로 쓰여짐

### Sonnet 지원시
- [ ] 명확하고 효율적
- [ ] 균형 잡힌 세부 수준
- [ ] 좋은 예시
- [ ] 합리적인 자유

### Opus 지원시
- [ ] 과도하게 설명되지 않음
- [ ] 추론 능력 신뢰
- [ ] 유연성 허용
- [ ] 독특한 것에 집중

---

## 품질 점수

체크마크를 세세요:

- **90-100%**: 우수, 공유 준비 완료
- **75-89%**: 양호, 공유 전 격차 해결
- **60-74%**: 개선 필요, 핵심 원칙 검토
- **< 60%**: 상당한 수정 필요

집중:
1. 핵심 품질 (가장 중요)
2. 테스트 (효과성 검증)
3. 구조 (발견 가능성 가능)
4. 기타 (다듬기)

# 기술 블로그 글쓰기 스타일 가이드

## 대상 독자

- 개발자 및 엔지니어
- 기술에 관심 있는 일반 독자
- 업계 동향을 파악하고자 하는 전문가

## 톤과 스타일

### 기본 원칙

1. **명확성**: 복잡한 개념을 쉽게 설명
2. **실용성**: 실제 적용 가능한 인사이트 제공
3. **전문성**: 깊이 있는 분석과 근거 제시
4. **친근함**: 딱딱하지 않은 대화체

### 문체

- 경어체 사용 ("~합니다", "~입니다")
- 불필요한 수식어 지양
- 능동태 선호
- 한 문장은 2줄 이내로

## 구조

### 1. 도입부 (Hook)

- 독자의 관심을 끄는 질문 또는 상황 제시
- 이 글에서 얻을 수 있는 것 명시
- 200자 이내

### 2. 본문

- 논리적 흐름으로 섹션 구성
- 각 섹션은 하나의 핵심 메시지
- 코드 예시는 주석과 함께
- 이미지/다이어그램 활용 권장

### 3. 결론

- 핵심 내용 요약 (3줄 이내)
- 실천 가능한 액션 아이템
- 추가 학습 리소스 링크

## 코드 블록

```language
// 주석으로 코드 설명
const example = "코드는 간결하게";
```

- 언어 명시 필수
- 실행 가능한 완전한 코드
- 핵심 부분 하이라이트

## 인용 및 출처

- 데이터는 출처 명시
- 외부 링크는 신뢰할 수 있는 소스만
- 인용문은 블록쿼트로 표시

> 인용문 예시입니다. - 출처

## SEO 고려사항

- 제목에 핵심 키워드 포함
- 소제목(H2, H3) 적극 활용
- 메타 설명 150자 이내
- 이미지 alt 텍스트 작성

## 품질 체크리스트

- [ ] 제목이 내용을 정확히 반영하는가
- [ ] 도입부가 흥미를 유발하는가
- [ ] 각 섹션이 논리적으로 연결되는가
- [ ] 코드 예시가 실행 가능한가
- [ ] 결론에 실천 가능한 내용이 있는가
- [ ] 출처가 명확히 표기되었는가
- [ ] 맞춤법과 문법이 정확한가

# 한국어 기술 글쓰기 가이드

범용 한국어 기술 글쓰기 원칙을 정리한 가이드.
블로그, 문서, 기술 자료 등 한국어로 작성하는 모든 콘텐츠에 적용 가능합니다.

---

## 1. 한국어 자연체 글쓰기 안티패턴

AI/번역체 표현을 자연스러운 한국어로 교체하세요.

| 금지 표현 | 자연스러운 대안 |
|----------|----------------|
| ~하는 것입니다 | ~해요, ~합니다, ~거든요 |
| ~할 수 있습니다 | ~돼요, ~됩니다 |
| ~하는 것이 중요합니다 | ~가 중요해요, ~해야 해요 |
| ~라고 할 수 있습니다 | ~예요, ~인 셈이죠 |
| ~를 통해 | ~로, ~해서 |
| ~에 대해 | ~를, ~관해 |
| ~함으로써 | ~하면, ~해서 |
| 이것은 ~입니다 | 이건 ~예요, ~거든요 |
| 그러므로 | 그래서, 그러니까 |
| 따라서 | 그래서, 결국 |
| 또한 | 그리고, 게다가, 또 |
| 하지만 | 근데, 그런데 |
| 매우, 상당히 | 진짜, 정말, 엄청 |
| ~되어집니다 | ~돼요 |
| 제공합니다 | 줘요, 드려요 |

### 자연스러움 점검 체크리스트

- "~하는 것입니다" 패턴이 없는가?
- "~할 수 있습니다" 대신 "~돼요/됩니다"를 사용했는가?
- "통해", "대해", "함으로써" 같은 번역체가 없는가?
- 문장이 짧고 읽기 쉬운가? (한 문장 2줄 이하)
- 실제 한국어 화자가 쓴 글처럼 읽히는가?

## 2. 기술 글쓰기 원칙

### 기본 원칙

1. **명확성**: 복잡한 개념을 쉽게 설명
2. **실용성**: 실제 적용 가능한 인사이트 제공
3. **전문성**: 깊이 있는 분석과 근거 제시
4. **친근함**: 딱딱하지 않은 대화체

### 문체

- 경어체 사용 ("~합니다", "~입니다")
- 불필요한 수식어 지양
- 능동태 선호
- 한 문장은 2줄 이내로
- 맞춤법과 문법을 정확히 지킴

### 구조

- 제목은 본문 내용을 정확히 반영하도록 작성

**도입부:**
- 독자의 관심을 끄는 질문 또는 상황 제시
- 이 글에서 얻을 수 있는 것 명시

**본문:**
- 논리적 흐름으로 섹션 구성
- 각 섹션은 하나의 핵심 메시지
- 코드 예시는 주석과 함께
- 이미지/다이어그램 활용 권장

**결론:**
- 핵심 내용 요약 (3줄 이내)
- 실천 가능한 액션 아이템
- 추가 학습 리소스 링크

### 코드 블록

- 언어 명시 필수
- 실행 가능한 완전한 코드
- 주석으로 핵심 부분 설명

### 인용 및 출처

- 데이터는 출처 명시
- 외부 링크는 신뢰할 수 있는 소스만
- 인용문은 블록쿼트로 표시

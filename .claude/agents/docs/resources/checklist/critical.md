## 마인드셋

- "What assumptions are being made?"
- "When would this fail?"
- "Is there a simpler explanation?"
- "Are we confusing correlation with causation?"

## 1. 가정 발굴 (Assumption Hunting)

- [ ] **CRT-1-001: 명시되지 않은 전제** - 작성자가 가정하고 있지만 명시하지 않은 전제는 무엇인가?
- [ ] **CRT-1-002: 독자 수준 가정** - 독자가 특정 지식을 알고 있다고 가정하는가?
- [ ] **CRT-1-003: 환경 가정** - 특정 환경/설정이 있다고 가정하는가?
- [ ] **CRT-1-004: 사용 사례 가정** - 특정 사용 사례에 최적화되어 다른 경우를 무시하는가?

## 2. 실패 시나리오 (Failure Scenarios)

- [ ] **CRT-2-001: 예외 처리** - 예외 상황이나 에러 케이스가 다뤯는가?
- [ ] **CRT-2-002: 경계 조건** - 극단적인 입력값(빈 값, 매우 큰 값, 특수문자) 처리는?
- [ ] **CRT-2-003: 경쟁 조건** - 동시성/타이밍 관련 문제가 있는가?
- [ ] **CRT-2-004: 리소스 한계** - 메모리, 디스크, 네트워크 제한 상황에서의 동작은?
- [ ] **CRT-2-005: 외부 의존성 실패** - 외부 서비스/API가 실패할 경우의 처리는?

## 3. 논리적 일관성 (Logical Consistency)

- [ ] **CRT-3-001: 모순 검출** - 문서 내 서로 모순되는 주장이 있는가?
- [ ] **CRT-3-002: 결론-근거** - 제시된 결론이 근거를 충분히 뒷받침하는가?
- [ ] **CRT-3-003: 순환 논리** - 순환 논법이 사용되고 있지 않은가?
- [ ] **CRT-3-004: 거짓 원인** - 상관관계를 인과관계로 오해하고 있지 않은가?
- [ ] **CRT-3-005: 흑백 논리** - 복잡한 문제를 지나치게 단순화하고 있지 않은가?

## 4. 과장된 주장 (Exaggerated Claims)

- [ ] **CRT-4-001: 검증되지 않은 성능** - "~배 빠르다" 등의 주장에 근거가 있는가?
- [ ] **CRT-4-002: 절대적 표현** - "항상", "절대", "완벽" 등의 표현이 적절한가?
- [ ] **CRT-4-003: 미래 예측** - 불확실한 미래를 확신에 차게 예측하는가?
- [ ] **CRT-4-004: 범용성 과장** - 특정 상황에만 적용되는 해결책을 보편화하는가?

## 5. 맥락 누락 (Missing Context)

- [ ] **CRT-5-001: 제약조건** - 해결책의 제약조건이 명시되어 있는가?
- [ ] **CRT-5-002: 대안 검토** - 다른 대안을 고려하고 비교했는가?
- [ ] **CRT-5-003: 트레이드오프** - 장점만 강조하고 단점을 숨기는가?
- [ ] **CRT-5-004: 역사적 맥락** - 왜 이런 접근법을 선택했는지 배경이 있는가?

## 6. 확인 편향 (Confirmation Bias)

- [ ] **CRT-6-001: 성공 사례만** - 성공 사례만 제시하고 실패 사례는 무시하는가?
- [ ] **CRT-6-002: 선택적 증거** - 불리한 증거는 제외하고 유리한 것만 선택하는가?
- [ ] **CRT-6-003: 반례 고려** - 반례 가능성을 고려했는가?

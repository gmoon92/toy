# 사용자 주도 커밋 지침

사용자가 명시적으로 커밋 메시지 검토를 요청할 때 사용하는 참조 지침입니다.

## Workflow (사용자 확인 포함)

### 1. 변경사항 분석

Git 변경사항을 확인하여 변경 범위를 파악합니다.

1. `git status`로 변경된 파일 확인
2. `git diff`로 **working tree 변경사항 확인**
3. 소스 코드 변경 분석 시 **가능하면 LSP를 우선 사용**

### 2. 커밋 메시지 작성

Conventional Commits 규약에 따라 커밋 메시지를 작성합니다.

**메시지 구조:**
```
<type>(scope): <description>

<body>

<footer>
```

### 3. 사용자 확인

**작성한 커밋 메시지를 사용자에게 보여주고 승인을 받습니다.**

```
다음과 같이 커밋하려고 합니다:

<type>(<scope>): <description>

- <change 1>
- <change 2>

승인하시겠습니까? (y/n)
```

### 4. 승인 후 커밋 실행

사용자가 승인하면 커밋을 실행합니다.

```bash
git commit <path> -m "<message>"
```

## 사용자 확인이 필요한 경우

- 처음 커밋하는 파일/모듈
- Breaking Change가 있는 경우
- 커밋 메시지에 불확실한 부분이 있는 경우
- 사용자가 명시적으로 검토를 요청한 경우

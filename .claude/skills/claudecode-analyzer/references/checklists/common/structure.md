# 문서 구조 공통 검증 (Common Structure Validation)

## 1. 마크다운 헤딩 구조

- [ ] 헤딩 레벨이 논리적으로 구성되었는지 (h1 → h2 → h3 순)
- [ ] h1 제목이 문서의 목적을 명확히 설명하는지
- [ ] 섹션 구분이 명확한지

## 2. 필수 섹션 존재 확인

### 모든 문서 공통

- [ ] 문서의 목적/역할을 설명하는 섹션
- [ ] 사용 방법/워크플로우 섹션
- [ ] 제약사항/주의사항 섹션 (해당되는 경우)

## 3. 가독성 및 일관성

- [ ] 일관된 용어 사용 (하나의 용어 선택하여 문서 전체에서 사용)
- [ ] 코드 블록에 적절한 언어 태그 사용
- [ ] 테이블이 가독성을 해치지 않는지
- [ ] 불필요한 ASCII 아트 다이어그램이 없는지

## 4. 파일 경로 및 참조

- [ ] 파일 참조가 Unix 스타일 경로(`/`)를 사용하는지 (Windows `\` 미사용)
- [ ] 상대 경로가 올바른지
- [ ] 외부 링크가 유효한지

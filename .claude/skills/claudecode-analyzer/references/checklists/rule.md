# Rule 검증 체크리스트

## Rule 특화 프론트매터 검증

- [ ] `paths`: 문자열 배열 또는 단일 문자열
- [ ] `paths`: 유효한 Glob 패턴 (`**/*.ts`, `src/**/*` 등)
- [ ] `description`: 200자 이하 권장

## Rule 특화 본문 내용 검증

### 규칙의 명확성
- [ ] 각 규칙이 구체적이고 실행 가능한지
- [ ] 모호한 표현("적절히", "가능하면") 대신 구체적 기준
- [ ] 규칙이 측정 가능한 기준을 포함 (예: "2-space indentation")

### 규칙의 구조
- [ ] 마크다운 헤더를 사용하여 규칙을 그룹화
- [ ] 관련 규칙이 섹션별로 묶여 있음
- [ ] 중요도 순서: 필수 → 권장 → 선택

### 예시 포함 여부
- [ ] 복잡한 규칙에 대해 코드 예시 포함
- [ ] "DO"와 "DON'T" 예시가 명확히 구분

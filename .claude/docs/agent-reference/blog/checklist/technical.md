## 1. 코드 예제 검증 (Code Examples)

- [ ] **TEC-1-001: 문법 오류 없음** - 모든 코드가 문법적으로 올바른가?
- [ ] **TEC-1-002: 실행 가능** - 코드가 복사-붙여넣기로 실행 가능한가?
- [ ] **TEC-1-003: 완전한 예제** - 필요한 import/include가 포함되어 있는가?
- [ ] **TEC-1-004: 예상 출력** - 예상되는 출력 결과가 명시되어 있는가?
- [ ] **TEC-1-005: 오류 처리** - 예외 상황에 대한 처리가 포함되어 있는가?

## 2. 용어 정의 (Terminology)

- [ ] **TEC-2-001: 미정의 용어 없음** - 모든 기술 용어가 적절히 정의되는가?
- [ ] **TEC-2-002: 용어 일관성** - 동일 개념에 대해 일관된 용어를 사용하는가?
- [ ] **TEC-2-003: 약어 설명** - 처음 사용되는 약어가 풀어서 설명되는가?
- [ ] **TEC-2-004: 번역 일관성** - 외래 용어의 한글 번역이 일관되는가?

## 3. 버전 정보 (Version Information)

- [ ] **TEC-3-001: 버전 명시** - 사용된 기술/라이브러리의 버전이 명시되어 있는가?
- [ ] **TEC-3-002: 최신 버전** - 참조하는 API/라이브러리가 최신 버전인가?
- [ ] **TEC-3-003: 버전 호환성** - 다른 버전과의 호환성이 명시되어 있는가?
- [ ] **TEC-3-004: 폐기 예정 기능** - 사용된 기능 중 폐기 예정(deprecated)인 것이 있는가?

## 4. 환경 설정 (Environment Setup)

- [ ] **TEC-4-001: 의존성 명시** - 필요한 의존성(패키지, 라이브러리)이 명시되어 있는가?
- [ ] **TEC-4-002: 설치 방법** - 의존성 설치 방법이 제공되는가?
- [ ] **TEC-4-003: 환경 변수** - 필요한 환경 변수 설정이 설명되는가?
- [ ] **TEC-4-004: 최소 요구사항** - 하드웨어/소프트웨어 최소 요구사항이 명시되어 있는가?
- [ ] **TEC-4-005: 권한 설정** - 필요한 권한이나 인증 설정이 설명되는가?

## 5. 명령어 및 설정 (Commands & Configuration)

- [ ] **TEC-5-001: 실행 가능한 명령어** - 제시된 명령어가 실제로 작동하는가?
- [ ] **TEC-5-002: 설정 파일 예시** - 설정 파일의 완전한 예시가 제공되는가?
- [ ] **TEC-5-003: 경로 처리** - 파일 경로가 적절히 처리되었는가 (절대/상대 경로)?
- [ ] **TEC-5-004: 플랫폼 고려** - OS별 차이점이 언급되어 있는가?

## 6. 보안 고려사항 (Security)

- [ ] **TEC-6-001: 민감 정보 노출** - 하드코딩된 비밀번호/키가 없는가?
- [ ] **TEC-6-002: SQL 인젝션** - 사용자 입력이 적절히 검증/이스케이프되는가?
- [ ] **TEC-6-003: XSS 방지** - 웹 관련 코드에서 XSS 취약점이 없는가?
- [ ] **TEC-6-004: 보안 모범사례** - 보안 관련 모범사례가 따르는가?

## 7. 외부 도구 검증 (External Verification)

- [ ] **TEC-7-001: Context7 검증** - Claude API, MCP, LSP 등 관련 기술은 Context7로 최신 문서와 대조하였는가?
- [ ] **TEC-7-002: 공식 문서 참조** - 공식 문서와 일치하는 정보인가?
- [ ] **TEC-7-003: 버전 호환성** - Claude Code 버전과 호환되는 내용인가?
- [ ] **TEC-7-004: 정확성 주석** - 예시/추정치는 "예시", "대략" 등으로 명시하였는가?

## 1. 명확성 (Clarity)

- [ ] **STD-1-001: 목적 명시** - 문서의 목적이 명확히 명시되어 있는가?
- [ ] **STD-1-002: 핵심 메시지** - 핵심 메시지가 뚜렷하게 강조되어 있는가?
- [ ] **STD-1-003: 제목 적절성** - 제목이 내용을 정확히 반영하는가?
- [ ] **STD-1-004: 요약 존재** - TL;DR 또는 요약 섹션이 있는가 (긴 문서의 경우)?

## 2. 완전성 (Completeness)

- [ ] **STD-2-001: 필수 섹션 포함** - 문서 유형에 필요한 모든 섹션이 포함되어 있는가?
- [ ] **STD-2-002: 배경 정보** - 독자가 이해하기 위한 충분한 배경이 제공되는가?
- [ ] **STD-2-003: 예시 포함** - 추상적 개념에 대한 구체적 예시가 있는가?
- [ ] **STD-2-004: 참조 정보** - 관련 링크나 참고 자료가 제공되는가?

## 3. 정확성 (Accuracy)

- [ ] **STD-3-001: 사실 검증** - 제시된 사실이 정확한가?
- [ ] **STD-3-002: 최신 정보** - 정보가 최신 상태를 유지하는가?
- [ ] **STD-3-003: 논리적 타당성** - 주장과 근거가 논리적으로 연결되는가?
- [ ] **STD-3-004: 오탈자 없음** - 맞춤법과 문법 오류가 없는가?

## 4. 일관성 (Consistency) & 톤앤매너 (Tone & Manner)

- [ ] **STD-4-001: 용어 통일** - 동일한 개념에 대해 일관된 용어를 사용하는가?
  - Skills vs 스킬, Hooks vs 훅 등 혼용되지 않는가?
- [ ] **STD-4-002: 서식 통일** - 헤딩, 목록, 코드 블록 등의 서식이 일관되는가?
- [ ] **STD-4-003: 인칭 통일** - 1인칭/2인칭/3인칭 사용이 일관되는가?
  - "사용자" vs "개발자" 등 대상 독자 지칭이 통일되는가?
- [ ] **STD-4-004: 어투 통일** - 문서 전체의 어투(격식체/비격식체)가 일관되는가?
  - 한국어 문서: "~합니다" vs "~해요" 혼용되지 않는가?
  - 영어 문서: "~ing" vs "~s" 혼용되지 않는가?
- [ ] **STD-4-005: 톤 일관성** - 안내/설명/경고 등 목적에 맞는 톤이 유지되는가?
- [ ] **STD-4-006: 대상 독자 일관성** - 문서 전체에서 동일한 수준의 독자를 가정하는가?

## 5. 가독성 (Readability)

- [ ] **STD-5-001: 문장 길이** - 문장이 너무 길지 않은가 (30단어 이하 권장)?
- [ ] **STD-5-002: 단락 길이** - 단락이 적절한 길이인가 (3-5문장 권장)?
- [ ] **STD-5-003: 전문 용어 설명** - 전문 용어가 적절히 설명되거나 정의되는가?
- [ ] **STD-5-004: 시각적 구분** - 헤딩, 목록, 강조 등으로 시각적 구분이 잘 되는가?

## 6. 구조 (Structure)

- [ ] **STD-6-001: 헤딩 계층** - 헤딩 레벨이 논리적으로 구성되어 있는가?
- [ ] **STD-6-002: 섹션 순서** - 섹션의 순서가 논리적인 흐름을 따르는가?
- [ ] **STD-6-003: 내용 균형** - 각 섹션의 중요도와 길이가 균형을 이루는가?
- [ ] **STD-6-004: 네비게이션** - 목차나 링크를 통한 네비게이션이 용이한가?

## 배경

`subagent-creator` 스킬 개선 작업(완료)에서 블루팀/레드팀 5라운드 토론 패턴을 사용했고,
이 패턴을 재사용 가능한 스킬로 추상화하기로 결정했다.

---

## 용도

- Claude 메타 문서 (스킬, 에이전트, 규칙, CLAUDE.md 등) 고도화
- 설계 단계 문서 작성
- 요구사항에 따른 기능 목록 추출
- 여러 회의를 거친 고도화된 문서 산출이 필요한 모든 경우

---

## 확정된 설계

### 스킬명
`workshop`

### 전체 흐름

```
Phase 0: 스무고개 인터뷰 (강제 — 건너뛸 수 없음)
    ↓
Phase 1: 심도 추론 문서 작성
    ↓
Phase 2: 블루팀/레드팀 토론 (기본 3라운드, 최대 5라운드)
    ↓
Phase 3: 모호한 부분 사용자 확인 → 파일 적용
```

---

### Phase 0 — 스무고개 인터뷰 (강제)

**목적**: 사용자가 말로 표현하지 않은 숨겨진 요구사항, 중요한 제약, 사고 방식을 끌어낸다.
단순한 요구사항 수집이 아니라 "왜"와 "경계"를 파악하는 단계.

**방식**:
- Claude가 3~5개 핵심 질문을 순차적으로 던진다 (스무고개 방식)
- 질문 예시 유형:
  - "이것이 실패한 경우는 어떤 상황인가요?" (실패 기준 파악)
  - "가장 중요하게 생각하는 제약 조건이 있나요?" (경계 파악)
  - "이 작업 후 기대하는 결과물이 어떤 형태인가요?" (출력 기준)
  - "현재 무엇이 가장 불만족스러운가요?" (핵심 문제 파악)
- 질문은 대상과 목적에 따라 Claude가 동적으로 생성
- 모든 답변이 나올 때까지 Phase 1으로 넘어가지 않는다

**강제화 방법**:
스킬 워크플로우에 "인터뷰 완료 전 다음 단계 진행 금지" 명시

---

### Phase 1 — 심도 추론 문서

**목적**: 인터뷰 결과 + Claude 분석을 합친 공통 기반 문서.
블루팀/레드팀이 이 문서를 기반으로 토론한다.
이 문서 없이는 토론이 시작되지 않는다.

**내용 구조**:
```markdown
# 심도 추론 문서

## 사용자 의도 (인터뷰 기반)
- 핵심 목적
- 중요 제약
- 실패 기준
- 기대 결과물

## Claude 분석
- 현재 상태 진단
- 발견된 문제/기회
- 핵심 의사결정 포인트

## 공통 합의 사항
- 토론에서 다룰 핵심 질문들
- 합의된 제약 조건
```

**저장 경로**: `.claude/docs/plans/<feature>/depth-reasoning.md`

---

### Phase 2 — 블루팀/레드팀 토론

**라운드 수**: 기본 3라운드, 레드팀 "조건부 합격" 판정 시 최대 5라운드로 자동 확장

```
R1 블루팀: 심도 추론 문서 기반 설계 제안
R2 레드팀: 치명적 문제 / 경고 / 제안 분류해 비판
R3 블루팀: R2 반영 개선 → 레드팀 최종 판정
  → 합격: 확정
  → 조건부 합격: R4 최종 확정으로 이동
  → 불합격: R3 재진행
R4 (조건부 합격 시): 조건 해결 + 최종 설계 확정
R5 (필요 시): 엣지 케이스 추가 검증
```

**에이전트 저장 경로**: `.claude/docs/plans/<feature>/round-N-{blue|red|final}.md`

---

### Phase 3 — 사용자 확인 후 적용

- 파일 변경 범위가 크거나 모호한 결정이 있으면 사용자에게 확인
- 명확한 경우 바로 적용
- 적용 체크리스트는 최종 설계 문서에 포함

## 미결 사항 (다음 세션에서 결정)

1. **인터뷰 질문 생성 방식**: 대상 파일을 먼저 읽고 동적으로 생성 vs 미리 정의된 질문 풀에서 선택
2. **심도 추론 문서 작성자**: Claude 혼자 작성 vs 인터뷰 중 실시간으로 같이 작성
3. **스킬 파일 위치**: `~/.claude/skills/workshop/` (사용자 전역) vs `.claude/skills/workshop/` (프로젝트)
4. **호출 방식**: `workshop` 스킬로만 호출 vs 슬래시 커맨드 `/workshop` 병행

## 평가 기준

### 1. 중복 섹션 검토

**목적**: 통합 과정에서 발생할 수 있는 내용 중복 확인

**검토 항목**:
- 동일한 개념이 여러 섹션에서 반복 설명되는지
- 유사한 예시가 중복되는지
- 서로 다른 문서 간 중복 내용 (문서 간 참조로 대체 가능 여부)

**수용 기준**:
- 의도적인 반복 (강조 목적)은 허용
- 불필요한 중복은 제거 또는 참조 링크로 대체

---

### 2. 기술적 정확성 검증

**목적**: Context7을 활용하여 기술 내용의 정확성 확인

**검증 대상**:
- Claude Code 동작 원리
- Context Window 관련 수치 (200K, 1M 등)
- Skills/Hooks 동작 메커니즘
- Agent 시스템 동작 방식
- LSP/MCP 관련 기술 내용

**검증 방법**:
- Context7 Claude Code 문서 조회
- 공식 문서와 교차 검증
- 최신 정보 반영 (2026-03-23 기준)

---

### 3. 섹션간 흐름 검토

**목적**: 문서 내 섹션 전환의 자연스러움 확인

**검토 항목**:
- 섹션 도입 문장의 적절성
- 이전 섹션과의 연결성
- 논리적 흐름의 연속성
- 예상치 못한 주제 전환 여부

**개선 방향**:
- 필요시 전환 문구 추가
- 섹션 순서 재배치 검토
- 개요(overview) 문구 강화

---

### 4. 시각화 자료 검토

**목적**: 과도한 시각화 자료의 적절성 평가

**검토 항목**:
- 표(Table)의 과다 사용 여부
- 다이어그램의 필요성
- 서술로 대체 가능한 시각화

**원칙**:
- **서술 우선**: 가능한 한 서술형 문장으로 작성
- **표는 최소화**: 필수적인 경우에만 사용
- **Mermaid 다이어그램**: 복잡한 흐름에서만 사용

---

### 5. 톤앤매너 통일성

**목적**: 3개 출처 문서의 다른 톤을 통일

**검토 항목**:
- 문체 (합니다 vs 해요 vs 한다)
- 명령/권장 표현 방식
- 경고/주의 표현 방식
- 독자 호칭 방식

**목표 톤**:
- **형식**: "~합니다" (격식체)
- **독자**: 개발자/기술자 대상
- **분위기**: 전문적이지만 접근하기 쉬운 톤

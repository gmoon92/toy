# MCP 기본 개념

MCP(Model Context Protocol)의 역할과 동작 원리를 개념 중심으로 정리한 문서입니다.

---

## 1. 기존 LLM의 동작 방식과 한계

Claude, Kimi, GPT와 같은 LLM(대형 언어 모델)은 기본적으로 **텍스트 입력을 받아 텍스트를 생성하는 모델**입니다.

즉, 다음과 같이 단순히 텍스트를 입력받고 출력하는 작업은 가능합니다.

- 질문에 답변
- 코드 생성
- 문서 작성

하지만 다음 작업은 **LLM이 직접 수행할 수는 없습니다.**

- 파일 수정
- 브라우저 조작
- 데이터베이스 조회
- API 호출

LLM은 실행 주체가 아니라, 실행을 "요청하는 텍스트"를 생성하는 모델이기 때문입니다.

## 2. 실행 문제를 해결하기 위한 확장

AI가 실제 작업을 수행하려면 **텍스트 생성 능력과 외부 프로그램 실행 능력을 연결해주는 구조**가 필요합니다.

이 문제를 해결하기 위해 등장한 것이 **MCP(Model Context Protocol)**입니다.

## 3. MCP란?

**MCP(Model Context Protocol)** 는 AI 모델이 외부 도구(Tool)와 통신할 수 있도록 정의된 표준 프로토콜입니다.

- **MCP**: 통신 규칙(Protocol)
- **MCP 서버**: 해당 규칙을 구현한 외부 실행 프로그램

MCP가 표준 프로토콜이기 때문에 다음과 같은 장점이 있습니다.

- 특정 모델에 종속되지 않음
- 모델 교체(Claude -> Kimi -> GPT 등)와 무관하게 동일한 구조 유지
- AI의 추론 결과를 실제 실행으로 연결하는 구조 제공

## 4. CLI 기반 MCP 동작 구조

실행 주체는 **CLI**이며, LLM은 **의사결정 및 명령 생성 역할**을 담당합니다.

```text
사용자
   ↓ (명령 입력)
CLI
   ↓ (프롬프트 전달)
LLM
   ↓ (Tool Call 생성)
CLI
   ↓ (MCP 프로토콜)
MCP Server (외부 프로그램)
   ↓
파일 / 브라우저 / DB / API
```

역할 분리는 다음과 같습니다.

- **LLM**: 실행 주체가 아닌 의사결정 주체, Tool Call 생성
- **CLI**: LLM 요청 해석 및 MCP 서버 실행
- **MCP 서버**: 실제 시스템 작업 수행

## 5. MCP 연결 시 가능한 변화

예를 들어 "이 코드의 문제점을 찾아서 수정해줘"라는 요청 기준으로 보면 다음과 같습니다.

| 구분     | MCP 미연결              | MCP 연결                           |
|--------|----------------------|----------------------------------|
| **작업** | 코드 분석 및 수정 방법 텍스트 설명 | 코드 분석 -> CLI가 MCP 호출 -> 실제 파일 수정 |
| **결과** | 설명과 제안까지만 가능         | 추론 -> 실행 -> 결과 알림까지 완료           |

MCP는 답변 품질 향상 기능이 아니라, LLM의 추론 결과를 실제 시스템 작업으로 이어지게 만드는 **실행 연결 구조**입니다.

# [XML 태그를 사용하여 프롬프트 구조화하기](https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/use-xml-tags)

---

> **참고**
>
> 이 팁들은 모든 Claude 모델에 광범위하게 적용되지만, 확장 사고(extended thinking) 모델에 특화된 프롬프팅 팁은 [여기](../08-prompt-engineering/13-extended-thinking-tips.md)에서 확인할 수 있습니다.

프롬프트에 컨텍스트, 지시사항, 예시와 같은 여러 구성 요소가 포함될 때, XML 태그는 게임 체인저가 될 수 있습니다. XML 태그는 Claude가 프롬프트를 더 정확하게 파싱하도록 도와주어 더 높은 품질의 출력을 생성합니다.

> **XML 팁**: `<instructions>`, `<example>`, `<formatting>`과 같은 태그를 사용하여 프롬프트의 다른 부분을 명확하게 구분하세요. 이를 통해 Claude가 지시사항을 예시나 컨텍스트와 혼동하는 것을 방지할 수 있습니다.

## XML 태그를 사용하는 이유는?

- **명확성:** 프롬프트의 다른 부분을 명확하게 구분하고 프롬프트가 잘 구조화되도록 보장합니다.
- **정확성:** Claude가 프롬프트의 일부를 잘못 해석하여 발생하는 오류를 줄입니다.
- **유연성:** 모든 것을 다시 작성하지 않고도 프롬프트의 일부를 쉽게 찾고, 추가하고, 제거하거나 수정할 수 있습니다.
- **파싱 가능성:** Claude가 출력에서 XML 태그를 사용하도록 하면 후처리를 통해 응답의 특정 부분을 추출하기가 더 쉬워집니다.

> **참고**
>
> Claude가 특별히 훈련된 정규적인 "최선의" XML 태그는 없지만, 태그 이름이 둘러싸고 있는 정보와 의미가 통하도록 하는 것을 권장합니다.

---

## 태깅 모범 사례

1. **일관성 유지**: 프롬프트 전체에서 동일한 태그 이름을 사용하고, 콘텐츠에 대해 이야기할 때 해당 태그 이름을 참조하세요 (예: `<contract> 태그의 계약서를 사용하여...`).
2. **태그 중첩**: 계층적 콘텐츠에 대해서는 태그를 `<outer><inner></inner></outer>`와 같이 중첩해야 합니다.

> **파워 유저 팁**: XML 태그를 멀티샷 프롬프팅(`<examples>`)이나 생각의 사슬(`<thinking>`, `<answer>`)과 같은 다른 기법과 결합하세요. 이를 통해 매우 구조화되고 고성능의 프롬프트를 만들 수 있습니다.

### 예시

#### 예시: 재무 보고서 생성

XML 태그가 없으면 Claude는 작업을 잘못 이해하고 필요한 구조나 톤과 맞지 않는 보고서를 생성합니다. 치환 후에도 Claude가 한 섹션(예: Q1 보고서 예시)이 끝나고 다른 섹션이 시작되는 지점을 잘못 이해할 가능성이 있습니다.

| 역할 | XML 태그 없음 | XML 태그 사용 |
| ---- | ----------- | ------------- |
| User | 당신은 AcmeCorp의 재무 분석가입니다. 투자자를 위한 Q2 재무 보고서를 생성하세요. 작년의 이 예시처럼 수익 성장, 이익 마진, 현금 흐름에 대한 섹션을 포함하세요: \{\{Q1_REPORT\}\}. 이 스프레드시트의 데이터 포인트를 사용하세요: \{\{SPREADSHEET_DATA\}\}. 보고서는 매우 간결하고, 요점 중심적이며, 전문적이고, 목록 형식이어야 합니다. 강점과 개선이 필요한 영역을 모두 강조해야 합니다. | 당신은 AcmeCorp의 재무 분석가입니다. 투자자를 위한 Q2 재무 보고서를 생성하세요.<br/><br/>AcmeCorp는 B2B SaaS 회사입니다. 우리 투자자들은 투명성과 실행 가능한 인사이트를 중요하게 생각합니다.<br/><br/>보고서에 다음 데이터를 사용하세요:\<data>\{\{SPREADSHEET_DATA\}\}\</data><br/><br/>\<instructions><br/>1. 수익 성장, 이익 마진, 현금 흐름 섹션을 포함하세요.<br/>2. 강점과 개선이 필요한 영역을 강조하세요.<br/>\</instructions><br/><br/>간결하고 전문적인 톤으로 작성하세요. 다음 구조를 따르세요:<br/>\<formatting_example>\{\{Q1_REPORT\}\}\</formatting_example> |
| Assistant | AcmeCorp 투자자를 위한 Q2 재무 보고서<br/><br/>존경하는 투자자 여러분,<br/><br/>B2B SaaS 시장에서 AcmeCorp의 지속적인 성장과 회복력을 보여주는 Q2 재무 보고서를 발표하게 되어 기쁩니다.<br/><br/>1. 수익 성장:<br/>Q2 매출은 1,520만 달러로 전년 대비 22% 증가했습니다. 이 성장은 주로 엔터프라이즈 고객 기반의 30% 확대에 의해 주도되었습니다. 그러나 SMB 부문은 5% 소폭 감소했으며, Q3에 타겟 마케팅 이니셔티브를 통해 이를 해결할 계획입니다.<br/><br/>2. 이익 마진:<br/>인프라 최적화로 인해 총 마진이 72%로 개선되었습니다 (Q1 대비 3% 증가). 그러나 EBITDA 마진은 18%로 하락했습니다 (2% 감소). 이는 곧 출시될 AI 기반 기능을 위한 R&D에 대규모 투자를 했기 때문입니다.<br/><br/>3. 현금 흐름:<br/>영업 현금 흐름은 410만 달러로 강력하게 유지되고 있습니다. 현금 보유액은 2,800만 달러로 현재 소진율 기준 15개월의 런웨이를 제공합니다. 고객 획득 비용이 20% 증가했는데, 더 효율적인 판매 전략을 통해 이를 해결하고 있습니다.<br/><br/>요약하면, AcmeCorp의 핵심 재무는 견고하게 유지되고 있습니다. 성장 투자와 신중한 현금 관리 사이의 균형을 유지하고 있습니다. #### 예시: 법률 계약서 분석

XML 태그가 없으면 Claude의 분석은 체계적이지 않고 핵심 포인트를 놓칩니다. 태그를 사용하면 법무 팀이 조치를 취할 수 있는 구조화되고 철저한 분석을 제공합니다.

| 역할 | XML 태그 없음 | XML 태그 사용 |
| ---- | ----------- | ------------- |
| User | 잠재적 위험과 책임에 대해 이 소프트웨어 라이선스 계약을 분석하세요: \{\{CONTRACT\}\}. 면책, 책임 제한, IP 소유권 조항에 집중하세요. 또한 비정상적이거나 우려되는 조항을 메모하세요. 참고로 우리의 표준 계약서는 다음과 같습니다: \{\{STANDARD_CONTRACT\}\}. 법무 팀을 위한 조사 결과 요약과 권장사항을 제공하세요. | 법적 위험과 책임에 대해 이 소프트웨어 라이선스 계약을 분석하세요.<br/><br/>우리는 핵심 데이터 인프라를 위해 이 계약을 고려 중인 다국적 기업입니다.<br/><br/>\<agreement><br/>\{\{CONTRACT\}\}\</agreement><br/><br/>참고용 우리의 표준 계약서입니다:<br/>\<standard_contract>\{\{STANDARD_CONTRACT\}\}\</standard_contract><br/><br/>\<instructions><br/>1. 다음 조항을 분석하세요:<br/>   - 면책<br/>   - 책임 제한<br/>   - IP 소유권<br/><br/>2. 비정상적이거나 우려되는 조항을 메모하세요.<br/><br/>3. 우리의 표준 계약서와 비교하세요.<br/><br/>4. 조사 결과를 \<findings> 태그로 요약하세요.<br/><br/>5. 실행 가능한 권장사항을 \<recommendations> 태그로 나열하세요.\</instructions> |
| Assistant | 제공된 소프트웨어 라이선스 계약은 몇 가지 우려스러운 측면을 제시합니다. 면책 조항이 지나치게 광범위하여 잠재적으로 우리를 상당한 책임에 노출시킬 수 있습니다. 책임 제한 섹션은 손해 배상을 단 500달러로 제한하는데, 이는 우리의 엔터프라이즈급 사용에는 부적절합니다. IP와 관련하여, 계약은 우리가 생성한 모든 수정 또는 통합에 대해 공급업체에 공동 소유권을 부여합니다.<br/><br/>요약하면, 이 계약은 우리의 최선의 이익에 부합하지 않습니다. 부당한 위험에 우리를 노출시키고, 실패 시 우리의 구제책을 제한하며, 지적 재산권을 훼손합니다. 이러한 조건을 재협상하거나 대체 공급업체를 찾는 것을 권장합니다. | \<findings><br/>1. 면책 (조항 8):<br/>   - 문제: 지나치게 광범위함. 우리의 사용으로 인해 발생하는 "모든 청구"에 대해 공급업체를 면책합니다.<br/>   - 위험: 공급업체의 과실이나 제품 결함에 대해서도 우리가 책임질 수 있습니다.<br/>   - 차이점: 우리의 표준은 계약 위반으로 인한 청구로 면책을 제한합니다.<br/><br/>2. 책임 제한 (조항 9):<br/>   - 문제: 직접 손해를 포함한 모든 손해를 500달러로 제한합니다.<br/>   - 위험: 엔터프라이즈 사용에 매우 부적절합니다. 중대한 실패는 수백만 달러의 비용이 들 수 있습니다.<br/>   - 차이점: 우리의 표준은 지불한 수수료의 12개월까지 직접 손해를 허용합니다.<br/><br/>3. IP 소유권 (조항 12):<br/>   - 문제: 우리의 수정/통합에 대해 공급업체에 공동 소유권을 부여합니다.<br/>   - 위험: 공급업체가 우리의 독점 알고리즘이나 데이터 인사이트를 사용하거나 판매할 수 있습니다.<br/>   - 차이점: 우리의 표준은 우리 작업물에 대한 완전한 소유권을 보유합니다.<br/><br/>4. 비정상적인 조항:<br/>   - 조항 5.3: 공급업체가 우리의 사용 데이터를 "모든 목적"으로 사용할 수 있습니다.<br/>   - 조항 7.2: 공급업체가 단 7일 전 통지로 "편의를 위해" 해지할 수 있습니다.<br/>   - 조항 11: 기본적인 상품성에 대한 보증조차 없는 어떠한 종류의 보증도 없습니다.<br/>\</findings><br/><br/>\<recommendations><br/>1. 이 계약을 거부하세요. 위험이 기업에 대한 혜택을 훨씬 초과합니다.<br/>2. 대안 제시:<br/>   - 면책: 계약 위반으로 인한 청구로 제한<br/>   - 책임: 직접 손해에 대해 12개월 수수료로 제한; 간접 손해에 대해 협상<br/>   - IP: 우리의 수정/통합에 대한 완전한 소유권<br/>   - 데이터 사용: 서비스 개선 목적으로만 제한<br/>   - 해지: 최소 90일 전 통지; 사유가 있는 경우에만<br/>   - 보증: 기본적인 상품성과 목적 적합성을 요구<br/>3. 공급업체가 거부하면 대체 공급자 또는 자체 솔루션을 탐색하세요.<br/>4. 높은 이해관계를 고려하여 계약 협상을 위해 법률 고문을 참여시키세요.<br/>\</recommendations> |

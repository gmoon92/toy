# [다국어 지원](https://platform.claude.com/docs/en/build-with-claude/multilingual-support)

Claude는 여러 언어에 걸친 작업에서 탁월한 성능을 발휘하며, 영어 대비 강력한 교차 언어 성능을 유지합니다.

---

## 개요

Claude는 강력한 다국어 기능을 보여주며, 특히 여러 언어에 걸친 제로샷(zero-shot) 작업에서 뛰어난 성능을 발휘합니다. 이 모델은 널리 사용되는 언어와 리소스가 적은 언어 모두에서 일관된 상대적 성능을 유지하여, 다국어 애플리케이션에 신뢰할 수 있는 선택지입니다.

아래에서 벤치마크된 언어 외에도 Claude는 많은 언어를 지원합니다. 특정 사용 사례와 관련된 모든 언어로 테스트해 보시기를 권장합니다.

> **참고**
> 이 지표는 전문 인간 번역가들에 의해 14개 추가 언어로 번역된 [MMLU (Massive Multitask Language Understanding)](https://en.wikipedia.org/wiki/MMLU) 영어 테스트 세트를 기반으로 하며, [OpenAI의 simple-evals 저장소](https://github.com/openai/simple-evals/blob/main/multilingual_mmlu_benchmark_results.md)에 문서화되어 있습니다. 이 평가에 인간 번역가를 사용함으로써 고품질 번역을 보장하며, 특히 디지털 리소스가 적은 언어에서 중요합니다.

## 모범 사례

다국어 콘텐츠 작업 시:

1. **명확한 언어 컨텍스트 제공**: Claude가 목표 언어를 자동으로 감지할 수 있지만, 원하는 입출력 언어를 명시적으로 지정하면 신뢰성이 향상됩니다. 유창성을 높이려면 Claude에게 "원어민처럼 관용적인 표현을 사용"하도록 프롬프트할 수 있습니다.
2. **원어 문자 체계 사용**: 최적의 결과를 위해 음역이 아닌 원어 문자로 텍스트를 제출하세요.
3. **문화적 맥락 고려**: 효과적인 의사소통은 순수한 번역을 넘어 문화적, 지역적 인식을 요구하는 경우가 많습니다.

또한 Claude의 성능을 더욱 개선하기 위해 일반적인 [프롬프트 엔지니어링 가이드라인](../08-prompt-engineering/01-overview.md)을 따르는 것을 권장합니다.

---

## 언어 지원 고려사항

- Claude는 표준 유니코드 문자를 사용하는 대부분의 세계 언어에서 입력을 처리하고 출력을 생성합니다.
- 성능은 언어에 따라 다르며, 특히 널리 사용되는 언어에서 강력한 기능을 제공합니다.
- 디지털 리소스가 적은 언어에서도 Claude는 의미 있는 기능을 유지합니다.

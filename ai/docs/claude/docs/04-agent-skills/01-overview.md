# [에이전트 스킬](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/overview)

에이전트 스킬(Agent Skills)은 Claude의 기능을 확장하는 모듈형 기능입니다. 각 스킬은 Claude가 관련 상황에서 자동으로 사용하는 지침, 메타데이터 및 선택적 리소스(스크립트, 템플릿)를 패키징합니다.

---

## 스킬을 사용하는 이유

스킬은 재사용 가능한 파일시스템 기반 리소스로, Claude에게 도메인별 전문 지식을 제공합니다: 워크플로우, 컨텍스트, 그리고 범용 에이전트를 전문가로 변환하는 모범 사례입니다. 프롬프트(일회성 작업을 위한 대화 수준의 지침)와 달리, 스킬은 필요에 따라 로드되며 여러 대화에서 동일한 지침을 반복적으로 제공할 필요가 없습니다.

**주요 이점**:
- **Claude 전문화**: 도메인별 작업에 맞춘 기능 제공
- **반복 감소**: 한 번 생성하면 자동으로 사용
- **기능 조합**: 스킬을 결합하여 복잡한 워크플로우 구축


> 에이전트 스킬의 아키텍처와 실제 적용 사례에 대한 심층 분석은 엔지니어링 블로그를 참고하세요: [Equipping agents for the real world with Agent Skills](https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills).


## 스킬 사용하기

Anthropic은 일반적인 문서 작업(PowerPoint, Excel, Word, PDF)을 위한 사전 구축된 에이전트 스킬을 제공하며, 사용자 정의 스킬을 직접 만들 수도 있습니다. 두 가지 모두 동일한 방식으로 작동합니다. Claude는 요청과 관련이 있을 때 자동으로 사용합니다.

**사전 구축된 에이전트 스킬**은 claude.ai의 모든 사용자와 Claude API를 통해 사용할 수 있습니다. 전체 목록은 아래 [사용 가능한 스킬](#사용-가능한-스킬) 섹션을 참조하세요.

**사용자 정의 스킬**을 사용하면 도메인 전문 지식과 조직 지식을 패키징할 수 있습니다. Claude의 제품 전반에서 사용할 수 있습니다: Claude Code에서 생성하거나, API를 통해 업로드하거나, claude.ai 설정에서 추가할 수 있습니다.


> **시작하기:**
> - 사전 구축된 에이전트 스킬의 경우: [빠른 시작 튜토리얼](../04-agent-skills/02-quickstart.md)을 참조하여 API에서 PowerPoint, Excel, Word 및 PDF 스킬 사용을 시작하세요
> - 사용자 정의 스킬의 경우: [에이전트 스킬 쿡북](https://platform.claude.com/cookbook/skills-notebooks-01-skills-introduction)을 참조하여 자신만의 스킬을 만드는 방법을 배우세요


## 스킬 작동 방식

스킬은 Claude의 VM 환경을 활용하여 프롬프트만으로는 불가능한 기능을 제공합니다. Claude는 파일시스템 액세스가 가능한 가상 머신에서 작동하므로, 스킬은 지침, 실행 가능한 코드 및 참고 자료가 포함된 디렉토리로 존재할 수 있으며, 새 팀 멤버를 위해 만드는 온보딩 가이드처럼 구성됩니다.

이 파일시스템 기반 아키텍처는 **점진적 공개(progressive disclosure)**를 가능하게 합니다: Claude는 컨텍스트를 미리 소비하는 대신, 필요에 따라 정보를 단계적으로 로드합니다.

### 세 가지 스킬 콘텐츠 유형, 세 가지 로딩 레벨

스킬은 각각 다른 시점에 로드되는 세 가지 유형의 콘텐츠를 포함할 수 있습니다:

### 레벨 1: 메타데이터 (항상 로드됨)

**콘텐츠 유형: 지침**. 스킬의 YAML 프론트매터는 발견 정보를 제공합니다:

```yaml
---
name: pdf-processing
description: PDF 파일에서 텍스트와 테이블을 추출하고, 양식을 작성하며, 문서를 병합합니다. PDF 파일을 다루거나 사용자가 PDF, 양식 또는 문서 추출을 언급할 때 사용합니다.
---
```

Claude는 시작 시 이 메타데이터를 로드하고 시스템 프롬프트에 포함시킵니다. 이 경량 접근 방식은 컨텍스트 패널티 없이 많은 스킬을 설치할 수 있음을 의미합니다. Claude는 각 스킬의 존재와 사용 시기만 알고 있습니다.

### 레벨 2: 지침 (트리거될 때 로드됨)

**콘텐츠 유형: 지침**. SKILL.md의 본문은 절차적 지식을 포함합니다: 워크플로우, 모범 사례 및 가이드:

````markdown
# PDF 처리

## 빠른 시작

pdfplumber를 사용하여 PDF에서 텍스트를 추출합니다:

```python
import pdfplumber

with pdfplumber.open("document.pdf") as pdf:
    text = pdf.pages[0].extract_text()
```

고급 양식 작성은 [FORMS.md](FORMS.md)를 참조하세요.
````

스킬의 설명과 일치하는 것을 요청하면, Claude는 bash를 통해 파일시스템에서 SKILL.md를 읽습니다. 그때서야 이 콘텐츠가 컨텍스트 윈도우에 들어갑니다.

### 레벨 3: 리소스 및 코드 (필요에 따라 로드됨)

**콘텐츠 유형: 지침, 코드 및 리소스**. 스킬은 추가 자료를 번들로 제공할 수 있습니다:

```
pdf-skill/
├── SKILL.md (주요 지침)
├── FORMS.md (양식 작성 가이드)
├── REFERENCE.md (상세 API 참조)
└── scripts/
    └── fill_form.py (유틸리티 스크립트)
```

**지침**: 특수화된 가이드와 워크플로우를 포함하는 추가 마크다운 파일(FORMS.md, REFERENCE.md)

**코드**: Claude가 bash를 통해 실행하는 실행 가능한 스크립트(fill_form.py, validate.py); 스크립트는 컨텍스트를 소비하지 않고 결정적 작업을 제공합니다

**리소스**: 데이터베이스 스키마, API 문서, 템플릿 또는 예제와 같은 참고 자료

Claude는 참조될 때만 이러한 파일에 액세스합니다. 파일시스템 모델은 각 콘텐츠 유형이 서로 다른 강점을 가짐을 의미합니다: 유연한 가이드를 위한 지침, 신뢰성을 위한 코드, 사실적 조회를 위한 리소스.

| 레벨 | 로드 시점 | 토큰 비용 | 콘텐츠 |
|-------|------------|------------|---------|
| **레벨 1: 메타데이터** | 항상 (시작 시) | 스킬당 ~100 토큰 | YAML 프론트매터의 `name`과 `description` |
| **레벨 2: 지침** | 스킬 트리거 시 | 5k 토큰 미만 | 지침과 가이드가 포함된 SKILL.md 본문 |
| **레벨 3+: 리소스** | 필요 시 | 사실상 무제한 | 컨텍스트에 콘텐츠를 로드하지 않고 bash를 통해 실행되는 번들 파일 |

점진적 공개는 언제든지 관련 콘텐츠만 컨텍스트 윈도우를 차지하도록 보장합니다.

### 스킬 아키텍처

스킬은 Claude가 파일시스템 액세스, bash 명령 및 코드 실행 기능을 가진 코드 실행 환경에서 실행됩니다. 다음과 같이 생각해보세요: 스킬은 가상 머신의 디렉토리로 존재하며, Claude는 컴퓨터의 파일을 탐색하는 데 사용하는 것과 동일한 bash 명령을 사용하여 상호 작용합니다.

![에이전트 스킬 아키텍처 - 스킬이 에이전트의 구성 및 가상 머신과 통합되는 방식을 보여줍니다](/docs/images/agent-skills-architecture.png)

**Claude가 스킬 콘텐츠에 액세스하는 방법:**

스킬이 트리거되면, Claude는 bash를 사용하여 파일시스템에서 SKILL.md를 읽어 지침을 컨텍스트 윈도우에 가져옵니다. 해당 지침이 다른 파일(FORMS.md 또는 데이터베이스 스키마와 같은)을 참조하면, Claude는 추가 bash 명령을 사용하여 해당 파일도 읽습니다. 지침이 실행 가능한 스크립트를 언급하면, Claude는 bash를 통해 이를 실행하고 출력만 받습니다(스크립트 코드 자체는 컨텍스트에 들어가지 않습니다).

**이 아키텍처가 가능하게 하는 것:**

**온디맨드 파일 액세스**: Claude는 각 특정 작업에 필요한 파일만 읽습니다. 스킬에 수십 개의 참조 파일이 포함될 수 있지만, 작업에 영업 스키마만 필요한 경우 Claude는 해당 파일만 로드합니다. 나머지는 0 토큰을 소비하며 파일시스템에 남아 있습니다.

**효율적인 스크립트 실행**: Claude가 `validate_form.py`를 실행할 때, 스크립트 코드는 컨텍스트 윈도우에 로드되지 않습니다. 스크립트의 출력("검증 통과" 또는 특정 오류 메시지와 같은)만 토큰을 소비합니다. 이는 Claude가 즉석에서 동등한 코드를 생성하는 것보다 훨씬 효율적입니다.

**번들 콘텐츠에 실질적 제한 없음**: 파일은 액세스될 때까지 컨텍스트를 소비하지 않으므로, 스킬에 포괄적인 API 문서, 대용량 데이터 세트, 광범위한 예제 또는 필요한 모든 참고 자료를 포함할 수 있습니다. 사용되지 않는 번들 콘텐츠에는 컨텍스트 패널티가 없습니다.

이 파일시스템 기반 모델이 점진적 공개를 작동하게 만드는 것입니다. Claude는 온보딩 가이드의 특정 섹션을 참조하듯이 스킬을 탐색하며, 각 작업에 필요한 것만 정확히 액세스합니다.

### 예제: PDF 처리 스킬 로딩

다음은 Claude가 PDF 처리 스킬을 로드하고 사용하는 방법입니다:

1. **시작**: 시스템 프롬프트에 포함됨: `PDF 처리 - PDF 파일에서 텍스트와 테이블 추출, 양식 작성, 문서 병합`
2. **사용자 요청**: "이 PDF에서 텍스트를 추출하고 요약해줘"
3. **Claude 호출**: `bash: read pdf-skill/SKILL.md` → 지침이 컨텍스트에 로드됨
4. **Claude 판단**: 양식 작성이 필요하지 않으므로 FORMS.md는 읽지 않음
5. **Claude 실행**: SKILL.md의 지침을 사용하여 작업 완료

![컨텍스트 윈도우로 로딩되는 스킬 - 스킬 메타데이터와 콘텐츠의 점진적 로딩을 보여줍니다](/docs/images/agent-skills-context-window.png)

다이어그램 설명:
1. 시스템 프롬프트와 스킬 메타데이터가 사전 로드된 기본 상태
2. Claude가 bash를 통해 SKILL.md를 읽어 스킬 트리거
3. Claude가 필요에 따라 FORMS.md와 같은 추가 번들 파일을 선택적으로 읽음
4. Claude가 작업 진행

이 동적 로딩은 관련 스킬 콘텐츠만 컨텍스트 윈도우를 차지하도록 보장합니다.

## 스킬이 작동하는 곳

스킬은 Claude의 에이전트 제품 전반에서 사용할 수 있습니다:

### Claude API

Claude API는 사전 구축된 에이전트 스킬과 사용자 정의 스킬을 모두 지원합니다. 두 가지 모두 동일하게 작동합니다: 코드 실행 도구와 함께 `container` 파라미터에 관련 `skill_id`를 지정합니다.

**전제 조건**: API를 통해 스킬을 사용하려면 세 가지 베타 헤더가 필요합니다:
- `code-execution-2025-08-25` - 스킬은 코드 실행 컨테이너에서 실행됩니다
- `skills-2025-10-02` - 스킬 기능을 활성화합니다
- `files-api-2025-04-14` - 컨테이너로/에서 파일을 업로드/다운로드하는 데 필요합니다

`skill_id`(예: `pptx`, `xlsx`)를 참조하여 사전 구축된 에이전트 스킬을 사용하거나, 스킬 API(`/v1/skills` 엔드포인트)를 통해 직접 생성하고 업로드합니다. 사용자 정의 스킬은 조직 전체에서 공유됩니다.

자세한 내용은 [Claude API로 스킬 사용하기](../04-agent-skills/05-using-skills-with-api.md)를 참조하세요.

### Claude Code

[Claude Code](https://code.claude.com/docs/en/overview)는 사용자 정의 스킬만 지원합니다.

**사용자 정의 스킬**: SKILL.md 파일이 있는 디렉토리로 스킬을 생성합니다. Claude가 자동으로 발견하고 사용합니다.

Claude Code의 사용자 정의 스킬은 파일시스템 기반이며 API 업로드가 필요하지 않습니다.

자세한 내용은 [Claude Code에서 스킬 사용하기](https://code.claude.com/docs/en/skills)를 참조하세요.

### Claude Agent SDK

[Claude Agent SDK](../05-agent-sdk/01-overview.md)는 파일시스템 기반 구성을 통해 사용자 정의 스킬을 지원합니다.

**사용자 정의 스킬**: `.claude/skills/`에 SKILL.md 파일이 있는 디렉토리로 스킬을 생성합니다. `allowed_tools` 구성에 `"Skill"`을 포함하여 스킬을 활성화합니다.

SDK의 스킬은 SDK가 실행될 때 자동으로 발견됩니다.

자세한 내용은 [SDK의 에이전트 스킬](https://platform.claude.com/docs/en/agent-sdk/skills)을 참조하세요.

### Claude.ai

[Claude.ai](https://claude.ai)는 사전 구축된 에이전트 스킬과 사용자 정의 스킬을 모두 지원합니다.

**사전 구축된 에이전트 스킬**: 이러한 스킬은 문서를 만들 때 이미 백그라운드에서 작동하고 있습니다. Claude는 설정 없이 자동으로 사용합니다.

**사용자 정의 스킬**: 설정 > 기능을 통해 zip 파일로 자신만의 스킬을 업로드합니다. 코드 실행이 활성화된 Pro, Max, Team 및 Enterprise 플랜에서 사용할 수 있습니다. 사용자 정의 스킬은 각 사용자 개인에게 적용되며, 조직 전체에서 공유되지 않고 관리자가 중앙에서 관리할 수 없습니다.

Claude.ai에서 스킬 사용에 대한 자세한 내용은 Claude 헬프 센터의 다음 리소스를 참조하세요:
- [스킬이란 무엇인가요?](https://support.claude.com/en/articles/12512176-what-are-skills)
- [Claude에서 스킬 사용하기](https://support.claude.com/en/articles/12512180-using-skills-in-claude)
- [사용자 정의 스킬 만드는 방법](https://support.claude.com/en/articles/12512198-creating-custom-skills)
- [스킬을 사용하여 Claude에게 작업 방식 가르치기](https://support.claude.com/en/articles/12580051-teach-claude-your-way-of-working-using-skills)

## 스킬 구조

모든 스킬은 YAML 프론트매터가 있는 `SKILL.md` 파일이 필요합니다:


## 보안 고려사항

신뢰할 수 있는 출처의 스킬만 사용할 것을 강력히 권장합니다: 직접 만들었거나 Anthropic에서 얻은 스킬. 스킬은 지침과 코드를 통해 Claude에게 새로운 기능을 제공하며, 이는 강력하지만 악의적인 스킬이 스킬의 명시된 목적과 일치하지 않는 방식으로 도구를 호출하거나 코드를 실행하도록 Claude를 지시할 수 있음을 의미합니다.


> 신뢰할 수 없거나 알 수 없는 출처의 스킬을 사용해야 하는 경우, 극도의 주의를 기울이고 사용 전에 철저히 감사하세요. Claude가 스킬을 실행할 때 가진 액세스 권한에 따라, 악의적인 스킬은 데이터 유출, 무단 시스템 액세스 또는 기타 보안 위험으로 이어질 수 있습니다.


**주요 보안 고려사항**:
- **철저한 감사**: 스킬에 번들로 제공되는 모든 파일을 검토하세요: SKILL.md, 스크립트, 이미지 및 기타 리소스. 예상치 못한 네트워크 호출, 파일 액세스 패턴 또는 스킬의 명시된 목적과 일치하지 않는 작업과 같은 비정상적인 패턴을 찾으세요
- **외부 소스는 위험합니다**: 외부 URL에서 데이터를 가져오는 스킬은 특히 위험하며, 가져온 콘텐츠에 악의적인 지침이 포함될 수 있습니다. 신뢰할 수 있는 스킬도 외부 의존성이 시간이 지남에 따라 변경되면 손상될 수 있습니다
- **도구 오용**: 악의적인 스킬은 해로운 방식으로 도구(파일 작업, bash 명령, 코드 실행)를 호출할 수 있습니다
- **데이터 노출**: 민감한 데이터에 액세스할 수 있는 스킬은 정보를 외부 시스템으로 유출하도록 설계될 수 있습니다
- **소프트웨어 설치처럼 취급**: 신뢰할 수 있는 출처의 스킬만 사용하세요. 민감한 데이터나 중요한 작업에 액세스할 수 있는 프로덕션 시스템에 스킬을 통합할 때는 특히 주의하세요

## 사용 가능한 스킬

### 사전 구축된 에이전트 스킬

다음 사전 구축된 에이전트 스킬을 즉시 사용할 수 있습니다:

- **PowerPoint (pptx)**: 프레젠테이션 생성, 슬라이드 편집, 프레젠테이션 콘텐츠 분석
- **Excel (xlsx)**: 스프레드시트 생성, 데이터 분석, 차트가 포함된 보고서 생성
- **Word (docx)**: 문서 생성, 콘텐츠 편집, 텍스트 서식 지정
- **PDF (pdf)**: 포맷된 PDF 문서 및 보고서 생성

이러한 스킬은 Claude API와 claude.ai에서 사용할 수 있습니다. API에서 사용을 시작하려면 [빠른 시작 튜토리얼](../04-agent-skills/02-quickstart.md)을 참조하세요.

### 사용자 정의 스킬 예제

사용자 정의 스킬의 전체 예제는 [스킬 쿡북](https://platform.claude.com/cookbook/skills-notebooks-01-skills-introduction)을 참조하세요.

## 제한 사항 및 제약 조건

이러한 제한 사항을 이해하면 스킬 배포를 효과적으로 계획하는 데 도움이 됩니다.

### 플랫폼 간 가용성

**사용자 정의 스킬은 플랫폼 간에 동기화되지 않습니다**. 한 플랫폼에 업로드된 스킬은 다른 플랫폼에서 자동으로 사용할 수 없습니다:

- Claude.ai에 업로드된 스킬은 API에 별도로 업로드해야 합니다
- API를 통해 업로드된 스킬은 Claude.ai에서 사용할 수 없습니다
- Claude Code 스킬은 파일시스템 기반이며 Claude.ai 및 API와 별개입니다

사용하려는 각 플랫폼에 대해 스킬을 별도로 관리하고 업로드해야 합니다.

### 공유 범위

스킬은 사용하는 위치에 따라 다른 공유 모델을 가집니다:
- **Claude.ai**: 개별 사용자만; 각 팀 멤버는 별도로 업로드해야 합니다
- **Claude API**: 워크스페이스 전체; 모든 워크스페이스 멤버가 업로드된 스킬에 액세스할 수 있습니다
- **Claude Code**: 개인용(`~/.claude/skills/`) 또는 프로젝트 기반(`.claude/skills/`); Claude Code 플러그인을 통해서도 공유할 수 있습니다

Claude.ai는 현재 사용자 정의 스킬의 중앙 집중식 관리자 관리 또는 조직 전체 배포를 지원하지 않습니다.

### 런타임 환경 제약 조건

스킬에 사용할 수 있는 정확한 런타임 환경은 사용하는 제품 플랫폼에 따라 다릅니다.

 - **Claude.ai**:
    - **가변적인 네트워크 액세스**: 사용자/관리자 설정에 따라 스킬은 전체, 부분 또는 네트워크 액세스가 없을 수 있습니다. 자세한 내용은 [파일 생성 및 편집](https://support.claude.com/en/articles/12111783-create-and-edit-files-with-claude#h_6b7e833898) 지원 문서를 참조하세요.
- **Claude API**:
    - **네트워크 액세스 없음**: 스킬은 외부 API 호출을 하거나 인터넷에 액세스할 수 없습니다
    - **런타임 패키지 설치 없음**: 사전 설치된 패키지만 사용할 수 있습니다. 실행 중에 새 패키지를 설치할 수 없습니다.
    - **사전 구성된 의존성만**: 사용 가능한 패키지 목록은 [코드 실행 도구 문서](../03-tools/05-code-execution-tool.md)를 확인하세요
- **Claude Code**:
    - **전체 네트워크 액세스**: 스킬은 사용자 컴퓨터의 다른 프로그램과 동일한 네트워크 액세스 권한을 가집니다
    - **전역 패키지 설치는 권장하지 않습니다**: 스킬은 사용자 컴퓨터에 간섭하지 않도록 로컬에서만 패키지를 설치해야 합니다

이러한 제약 조건 내에서 작동하도록 스킬을 계획하세요.

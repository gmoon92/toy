# [환각 줄이기](https://platform.claude.com/docs/en/test-and-evaluate/strengthen-guardrails/reduce-hallucinations)

---

Claude와 같은 가장 진보된 언어 모델조차도 때때로 사실과 다르거나 주어진 맥락과 일치하지 않는 텍스트를 생성할 수 있습니다. "환각(hallucination)"이라고 알려진 이 현상은 AI 기반 솔루션의 신뢰성을 저해할 수 있습니다.
이 가이드에서는 환각을 최소화하고 Claude의 출력이 정확하고 신뢰할 수 있도록 보장하는 기법들을 살펴보겠습니다.

## 기본 환각 최소화 전략

- **Claude가 "모른다"고 말할 수 있도록 허용하기:** Claude에게 불확실성을 인정할 수 있는 권한을 명시적으로 부여하세요. 이 간단한 기법만으로도 잘못된 정보를 대폭 줄일 수 있습니다.

<details>
<summary>예시: 인수합병 보고서 분석</summary>

| 역할 | 내용 |
| ---- | ------- |
| User | 당신은 우리의 M&A 자문입니다. ExampleCorp의 AcmeCo 인수에 관한 이 보고서를 분석해주세요.<br/><br/>\<report><br/>\{\{REPORT}}<br/>\</report><br/><br/>재무 전망, 통합 리스크, 규제 장애물에 초점을 맞춰주세요. 어떤 측면에 대해 확신이 없거나 보고서에 필요한 정보가 부족하다면, "이를 확신 있게 평가할 충분한 정보가 없습니다"라고 말하세요. |
</details>

- **사실 근거를 위해 직접 인용 사용하기:** 긴 문서(>20K 토큰)와 관련된 작업의 경우, Claude가 작업을 수행하기 전에 먼저 원문 그대로의 인용문을 추출하도록 요청하세요. 이는 실제 텍스트에 응답을 근거하게 하여 환각을 줄입니다.

<details>
<summary>예시: 데이터 개인정보 보호 정책 감사</summary>

| 역할 | 내용 |
| ---- | ------- |
| User | 당신은 우리의 데이터 보호 책임자입니다. GDPR 및 CCPA 준수를 위해 이 업데이트된 개인정보 보호 정책을 검토해주세요.<br/>\<br/>\{\{POLICY}}<br/>\</policy><br/><br/>1. GDPR 및 CCPA 준수와 가장 관련성이 높은 정책의 정확한 인용문을 추출하세요. 관련 인용문을 찾을 수 없다면 "관련 인용문을 찾을 수 없습니다"라고 명시하세요.<br/><br/>2. 인용문을 사용하여 이러한 정책 섹션의 준수 여부를 분석하고, 번호로 인용문을 참조하세요. 추출된 인용문만을 기반으로 분석하세요. |
</details>

- **인용을 통한 검증**: Claude의 응답을 감사 가능하게 만들기 위해 각 주장에 대해 인용문과 출처를 명시하도록 하세요. 또한 Claude가 응답을 생성한 후 각 주장을 뒷받침하는 인용문을 찾아 검증하도록 할 수도 있습니다. 인용문을 찾을 수 없다면 해당 주장을 철회해야 합니다.

<details>
<summary>예시: 제품 출시 보도자료 작성</summary>

| 역할 | 내용 |
| ---- | ------- |
| User | 이 제품 브리핑과 시장 보고서의 정보만을 사용하여 새로운 사이버 보안 제품인 AcmeSecurity Pro에 대한 보도자료를 작성하세요.<br/>\<documents><br/>\{\{DOCUMENTS}}<br/>\</documents><br/><br/>작성 후, 보도자료의 각 주장을 검토하세요. 각 주장에 대해 이를 뒷받침하는 문서의 직접 인용문을 찾으세요. 주장을 뒷받침하는 인용문을 찾을 수 없다면, 보도자료에서 해당 주장을 제거하고 제거된 위치를 빈 대괄호 []로 표시하세요. |
</details>

## 고급 기법

- **사고 과정 검증**: 최종 답변을 제공하기 전에 Claude에게 단계별로 추론을 설명하도록 요청하세요. 이를 통해 잘못된 논리나 가정을 드러낼 수 있습니다.

- **Best-of-N 검증**: 동일한 프롬프트로 Claude를 여러 번 실행하고 출력을 비교하세요. 출력 간의 불일치는 환각을 나타낼 수 있습니다.

- **반복적 개선**: Claude의 출력을 후속 프롬프트의 입력으로 사용하여 이전 진술을 검증하거나 확장하도록 요청하세요. 이를 통해 불일치를 포착하고 수정할 수 있습니다.

- **외부 지식 제한**: 제공된 문서의 정보만 사용하고 일반 지식을 사용하지 않도록 Claude에게 명시적으로 지시하세요.


> 이러한 기법들이 환각을 크게 줄이지만, 완전히 제거하지는 못한다는 점을 기억하세요. 특히 중요한 결정의 경우 핵심 정보를 항상 검증하세요.
